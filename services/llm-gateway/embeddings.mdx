## Endpoint

```
POST v1/embeddings
```

Base URL: `{API_URL}/workspaces/slug:llm-gateway/webhooks/v1/embeddings`

## Request

```json
{
  "model": "text-embedding-3-small",
  "input": "The quick brown fox jumps over the lazy dog"
}
```

### Parameters

| Param | Type | Description |
|-------|------|-------------|
| `model` | string | Embedding model identifier |
| `input` | string/array | Text(s) to embed |
| `dimensions` | integer | Output dimensions (if model supports it) |

### Batch Input

```json
{
  "model": "text-embedding-3-small",
  "input": [
    "First text to embed",
    "Second text to embed",
    "Third text to embed"
  ]
}
```

## Response

```json
{
  "object": "list",
  "model": "text-embedding-3-small",
  "data": [
    {
      "object": "embedding",
      "index": 0,
      "embedding": [0.0023, -0.0091, 0.0156, ...]
    }
  ],
  "usage": {
    "prompt_tokens": 8,
    "total_tokens": 8
  }
}
```

## Authentication

Same as [chat completions](/services/llm-gateway/chat-completions) — supports both API key (`iak_*`) and workspace JWT.

For API key access, the `llm:embeddings:use` permission is required.

## Usage

Embeddings are primarily used by:

- **[Storage](/services/storage/overview)** — Generating embeddings for RAG document chunks during indexing
- **[Memories](/tools/memories)** — Generating embeddings for memory storage and recall
- **Agent Factory** — Embedding user queries for knowledge base search

All these workspaces call the embeddings endpoint via workspace JWT through their `_llm` helper automation.

## Architecture Note

Unlike chat completions (which resolve providers via `_resolve-provider` and make direct HTTP calls), embeddings delegate to the **OpenAI app import** (`OpenAI.embeddings`). The OpenAI app handles provider routing, batching, and parallel processing internally.

Batch configuration can come from the model catalog's `provider_config` fields (`batch_size`, `batch_parallel`) or defaults to `batchSize: 96` and `batchParallel: 5`.

## Default Model

The default embedding model is configured in `index.yml`:

```yaml
config:
  value:
    defaults:
      embeddings: text-embedding-3-small
```

The default dimensions vary by model (e.g., 1536 for `text-embedding-3-small`).
