## Endpoint

```
POST v1/embeddings
```

Base URL: `{API_URL}/workspaces/slug:llm-gateway/webhooks/v1/embeddings`

## Request

```json
{
  "model": "text-embedding-3-large",
  "input": "The quick brown fox jumps over the lazy dog"
}
```

### Parameters

| Param | Type | Description |
|-------|------|-------------|
| `model` | string | Embedding model identifier |
| `input` | string/array | Text(s) to embed |
| `dimensions` | integer | Output dimensions (if model supports it) |

### Batch Input

```json
{
  "model": "text-embedding-3-large",
  "input": [
    "First text to embed",
    "Second text to embed",
    "Third text to embed"
  ]
}
```

## Response

```json
{
  "object": "list",
  "model": "text-embedding-3-large",
  "data": [
    {
      "object": "embedding",
      "index": 0,
      "embedding": [0.0023, -0.0091, 0.0156, ...]
    }
  ],
  "usage": {
    "prompt_tokens": 8,
    "total_tokens": 8
  }
}
```

## Authentication

Same as [chat completions](/services/llm-gateway/chat-completions) — requires an `Authorization` header. Supports both API key and cross-workspace calls.

## Defaults Endpoint

Use `GET v1/defaults` to retrieve the currently configured default models, including embedding model metadata:

```json
{
  "embeddings": {
    "model_id": "text-embedding-3-large",
    "dimensions": 3072,
    "supported_dimensions": [256, 1024, 3072]
  },
  "completions": {
    "model_id": "gpt-5-chat-latest"
  },
  "image_generation": {
    "model_id": "vertex-imagen-4.0"
  }
}
```

The defaults endpoint resolves governance overrides — if the organization has a custom `default_embedding_model` configured in AI Governance, it will be returned instead of the platform default.

## Usage

Embeddings are primarily used by:

- **[Storage](/services/storage/overview)** — Generating embeddings for RAG document chunks during indexing
- **[Memories](/tools/memories)** — Generating embeddings for memory storage and recall

These workspaces call the embeddings endpoint via cross-workspace calls through their `_llm` helper automation. Governance controls (model access) apply to embedding requests the same way as chat completions.

## Architecture Note

Embeddings delegate to the **OpenAI app import** (`OpenAI.embeddings`), which handles provider routing, batching, and parallel processing internally. This is the same app used by chat completions (`OpenAI.chat-completion`).

Batch configuration can come from the model catalog's `provider_config` fields (`batch_size`, `batch_parallel`) or defaults to `batchSize: 96` and `batchParallel: 5`.

## Default Model

The default embedding model is configured in `index.yml`:

```yaml
config:
  value:
    defaults:
      embeddings: text-embedding-3-large
```

The default dimensions depend on the model (e.g., 3072 for `text-embedding-3-large`). Use the `v1/defaults` endpoint to retrieve the current model's dimensions and supported dimension options.
