The Prompt Library provides curated prompt templates for AI agents. It implements both the MCP **Prompts** protocol (for retrieving prompts) and the MCP **Tools** protocol (for searching prompts).

**Workspace:** `prompt-library` (11 automations)

## MCP Endpoint

```
POST {API_URL}/workspaces/slug:prompt-library/webhooks/mcp
```

**Protocol:** `2025-06-18`
**Capabilities:** `{ prompts: { listChanged: false }, tools: {} }`

## MCP Methods

### List Prompts

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "prompts/list",
  "params": {
    "cursor": "0"
  }
}
```

Returns paginated prompt list (50 per page). Cursor is the page offset.

### Get Prompt

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "prompts/get",
  "params": {
    "name": "market-analysis",
    "arguments": {
      "industry": "technology",
      "period": "Q3 2025"
    }
  }
}
```

Returns the full prompt with a `_meta` extension:

```json
{
  "result": {
    "messages": [
      {
        "role": "user",
        "content": {
          "type": "text",
          "text": "Analyze the technology market for Q3 2025..."
        }
      }
    ],
    "_meta": {
      "tool_choice": "bing_search",
      "suggested_tools": ["bing_search", "knowledge_base"],
      "category": "Learn",
      "tags": ["market", "analysis"]
    }
  }
}
```

The `_meta` extension is non-standard MCP — it allows prompts to declare:
- `tool_choice` — Forces a specific tool when using this prompt
- `suggested_tools` — Recommends tools to activate

### Search Prompts (via Tools)

```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "tools/call",
  "params": {
    "name": "prompt_library",
    "arguments": {
      "query": "data analysis",
      "category": "Learn"
    }
  }
}
```

## Prompt Structure

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Unique slug identifier |
| `title` | string | Display title |
| `description` | string | What the prompt does |
| `category` | string | Category (Write, Learn, Code, Coach, Brainstorm) |
| `tags` | array | Search tags |
| `content` | string | Prompt template (supports `{{argument}}` placeholders) |
| `arguments` | array | Input parameters with name, description, required |
| `tool_choice` | string | Force a specific tool |
| `suggested_tools` | array | Recommended tools |
| `enabled` | boolean | Whether the prompt is active |

## Seeded Content

The `seed-prompts` automation inserts 25 enterprise prompts across 5 categories:

| Category | Count | Examples |
|----------|-------|---------|
| Write | 5 | Email drafting, report writing, summarization |
| Learn | 5 | Market analysis, concept explanation, research |
| Code | 5 | Code review, debugging, documentation |
| Coach | 5 | Interview prep, presentation feedback, goal setting |
| Brainstorm | 5 | Strategy ideation, problem solving, creative thinking |

Some prompts reference tools:
- `tool_choice: bing_search` — For web research prompts
- `suggested_tools: [bing_search, knowledge_base]` — For research-heavy prompts

## Frontend Integration

The SecureChat frontend queries the Prompt Library to populate the prompt picker. When a user selects a prompt:

1. Frontend calls `prompts/get` with any required arguments
2. The prompt's `messages` are used to pre-fill the conversation
3. If `tool_choice` is set, that tool is activated
4. If `suggested_tools` are set, they're shown as recommendations

## Data Storage

Prompts are stored in the `prompt_library` collection:

| Field | Type | Indexed |
|-------|------|---------|
| `name` | text | Yes (unique) |
| `category` | text | Yes |
| `enabled` | boolean | Yes |

## Security

Prompt Library uses a stricter security model than most workspaces:
- No user-session-based event access rules
- Only `editor` and `workspace` (API key) roles can access
- No public page read rules
