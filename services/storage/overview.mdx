Storage is the backend service for all document and knowledge management operations. It handles file uploads, vector store management, document indexing (chunking + embedding), and skill configuration.

**Workspace:** `storage`

| `skills` | Search skill configurations |

## Installed Providers

Storage imports vector provider workspaces as apps. Three ship by default:

| App Slug | Workspace | Purpose |
|----------|-----------|---------|
| `VectorElasticsearch` | `vector-elasticsearch` | Elasticsearch dense_vector |
| `VectorOpensearch` | `vector-opensearch` | OpenSearch k-NN |
| `VectorMock` | `vector-mock` | Testing (fake results) |

Additional providers (Redis, PgVector, Qdrant, Pinecone) are available via the vector router but require separate workspace setup.

The active provider is selected by `config.default_vector_provider` (default: `elasticsearch`).

## Vector Router

The `_vector-router` dispatches operations to the correct provider. All providers implement the same interface ([see Vector Stores](/tools/vector-stores)), making them interchangeable.

## Indexing Pipeline

When a file is added to a vector store:

1. **Parse** â€” Extract text from the file (PDF, DOCX, etc.) via the Crawler/Parser
2. **Chunk** â€” Split text into chunks (`_chunk-text`)
3. **Embed** â€” Generate embeddings via [LLM Gateway](/services/llm-gateway/embeddings)
4. **Upsert** â€” Store chunks + embeddings in the vector provider

The indexing is triggered by:
- `POST v1/vector_stores/:id/files` â€” Add a file
- `POST v1/vector_stores/:id/files/:file_id/reindex` â€” Re-index a file
- Event `file.index_requested` â€” Automated indexing

## Authentication

Storage supports two auth modes:

| Mode | Use Case |
|------|----------|
| **Workspace JWT** | Called by Agent Factory, other workspaces. Trusts `body.user_id` and `body.orgSlug` |
| **User session** | Direct frontend access |

Storage is primarily accessed through workspace-to-workspace calls.

## Configuration

| Setting | Default | Description |
|---------|---------|-------------|
| `default_vector_provider` | `elasticsearch` | Vector backend |
| `default_embedding_model` | `text-embedding-3-small` | Embedding model |
| `default_embedding_dimensions` | 1536 | Embedding dimensions |
| `vector_store_index_prefix` | (env-specific) | Index name prefix |
| `user_data_expires_after` | 1800 | Temp data TTL (seconds) |
