---
title: "Skills"
description: "Reusable search configurations for agents"
---

Skills are predefined search configurations that define how an agent queries knowledge bases. They encapsulate vector store selection, search parameters, and result formatting into a reusable unit.

## Endpoints

```
GET    v1/skills              # List skills
POST   v1/skills              # Create skill
GET    v1/skills/:skill_id    # Get skill
PUT    v1/skills/:skill_id    # Update skill
DELETE v1/skills/:skill_id    # Delete skill
```

## Create Skill

```json
{
  "name": "Company Docs Search",
  "description": "Search company documentation with high relevance threshold",
  "vector_store_ids": ["vs_abc", "vs_def"],
  "config": {
    "top_k": 5,
    "min_score": 0.75,
    "reranking": false,
    "scope_filter": "org:{{orgSlug}}"
  }
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Skill name |
| `description` | string | What this skill searches |
| `vector_store_ids` | array | Vector stores to search |
| `config.top_k` | integer | Number of results to return |
| `config.min_score` | float | Minimum similarity threshold |
| `config.reranking` | boolean | Enable result reranking |
| `config.scope_filter` | string | Scope template (supports variables) |

## Usage in Agent Factory

Skills are referenced when configuring the `knowledge_search` system tool on an agent:

```json
{
  "type": "file_search",
  "name": "knowledge_base",
  "config": {
    "skill_id": "skill_xyz"
  }
}
```

When the agent calls `knowledge_search`, Agent Factory resolves the skill configuration and calls Storage's search endpoint with the appropriate parameters.

## Multi-Store Search

A skill can search across **multiple vector stores** simultaneously. Results from all stores are merged and ranked by similarity score:

```
Query ──► Skill (vs_abc, vs_def)
              │
              ├──► Search vs_abc ──► [results...]
              │
              └──► Search vs_def ──► [results...]
                                        │
                                        ▼
                                   Merge + Sort by score
                                        │
                                        ▼
                                   Top-K results
```

## Scope Templates

The `scope_filter` supports template variables:

| Variable | Description |
|----------|-------------|
| `{{orgSlug}}` | Current organization |
| `{{userId}}` | Current user ID |
| `{{agentId}}` | Current agent ID |

This enables dynamic scoping — the same skill produces different results based on who is using it.
