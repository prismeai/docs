---
title: "Skills"
description: "Reusable search configurations for agents"
---

Skills are stored configurations that define reusable instructions and tool setups for agents. They encapsulate what an agent should do and which tools it should use into a reusable unit.

## Endpoints

```
GET    v1/skills              # List skills
POST   v1/skills              # Create skill
GET    v1/skills/:skill_id    # Get skill
PATCH  v1/skills/:skill_id    # Update skill
DELETE v1/skills/:skill_id    # Delete skill
```

## Create Skill

```json
{
  "name": "Company Docs Assistant",
  "description": "Helps users search and understand company documentation",
  "instructions": "You are a helpful assistant that searches company docs. Always cite sources.",
  "tools": [
    { "type": "file_search", "name": "knowledge_base" }
  ]
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Skill name |
| `description` | string | No | What this skill does |
| `instructions` | string | Yes | System instructions for the agent |
| `tools` | JSON | No | Tool configurations to use with this skill |

## Usage

Skills are stored configurations that can be referenced by other workspaces when setting up agents. They provide a way to standardize agent behavior across different contexts by defining instructions and tool configurations in one place.
