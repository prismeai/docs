The Bing Web Search workspace provides web search capabilities for AI agents via the Azure AI Foundry Bing Grounding API. It implements the MCP protocol for seamless agent integration.

**Workspace:** `tools-search-bing`

## MCP Endpoint

```
POST {API_URL}/workspaces/slug:tools-search-bing/webhooks/bing-search/mcp
```

**Protocol:** `2025-06-18`
**Capabilities:** `{ tools: {} }`

## Tool Definition

The workspace exposes a single MCP tool:

```json
{
  "name": "bing_search",
  "description": "Search the web using Bing",
  "inputSchema": {
    "type": "object",
    "properties": {
      "query": {
        "type": "string",
        "description": "Search query"
      },
      "num_results": {
        "type": "number",
        "description": "Number of results (default: 5)"
      },
      "market": {
        "type": "string",
        "description": "Market code (default: en-US)"
      },
      "freshness": {
        "type": "string",
        "description": "Result freshness: Day, Week, Month"
      }
    },
    "required": ["query"]
  }
}
```

## How It Works

The search flow uses Azure AI Foundry's assistant API with Bing Grounding:

```mermaid
flowchart TD
    MCP[MCP tools/call] --> TOKEN[Get Azure AD Token\nOAuth2 client credentials\ncached up to 55 min]
    TOKEN --> ASST[Create Azure AI Assistant\nwith Bing Grounding tool]
    ASST --> THREAD[Create Thread + Message\nPost user query]
    THREAD --> RUN[Run Assistant\nPoll for completion — max 30s]
    RUN --> EXTRACT[Extract Response + Cleanup\nReturn results & delete assistant]
```

## Azure Configuration

The workspace requires Azure AI Foundry credentials:

| Config Key | Description |
|------------|-------------|
| `azure_tenant_id` | Azure AD tenant ID |
| `azure_client_id` | Azure AD application (client) ID |
| `azure_client_secret` | Azure AD client secret |
| `foundry_base_url` | Azure AI Foundry endpoint URL |
| `foundry_api_version` | API version |

Azure AD tokens are cached in the session for up to `token_cache_ttl` seconds (default: 3300s / 55 minutes).

## Security

Bing Search uses standard session-based event rules. It includes `editor`, `viewer`, and `workspace` (API key) roles. The workspace is primarily accessed via MCP calls from Agent Factory.

## Integration

The Bing Search tool can be registered in the [Capabilities catalog](/services/capabilities/overview) with its MCP server URL.

When an agent has `bing_search` configured, Agent Factory calls this MCP endpoint during tool execution in the [agentic loop](/products/agent-factory/agentic-loop).
