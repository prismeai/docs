Prisme.ai implements rate limiting to ensure platform stability and fair usage across all users.

## Rate Limit Overview

<Tabs>
  <Tab title="API Endpoints">
    Most API endpoints have rate limits tracked per authenticated user or API key:

    <CardGroup cols="3">
      <Card title="Standard APIs" icon="code">
        100 requests per minute
      </Card>
      <Card title="Write Operations" icon="pen-to-square">
        60 requests per minute
      </Card>
      <Card title="Search/List" icon="magnifying-glass">
        30 requests per minute
      </Card>
    </CardGroup>

    <Info>
      Specific endpoints may have custom limits based on resource intensity.
    </Info>
  </Tab>
  <Tab title="Runtime Automations">
    While executing workspace automations, the Runtime enforces four distinct rate limits:

    <CardGroup cols="2">
      <Card title="Automations Execution" icon="play">
        - Default: 100 automations/second
        - Burst: 400 automations
        - Env var: `RATE_LIMIT_AUTOMATIONS`
        - Workspace secret: `prismeai_ratelimit_automations`
      </Card>
      <Card title="Event Emits" icon="bell">
        - Default: 30 emits/second
        - Burst: 100 emits
        - Env var: `RATE_LIMIT_EMITS`
        - Workspace secret: `prismeai_ratelimit_emits`
      </Card>
      <Card title="HTTP Fetches" icon="globe">
        - Default: 50 fetches/second
        - Burst: 200 fetches
        - Env var: `RATE_LIMIT_FETCHS`
        - Workspace secret: `prismeai_ratelimit_fetchs`
      </Card>
      <Card title="Repeat Loops" icon="repeat">
        - Default: 1000 iterations/second
        - Burst: 4000 iterations
        - Env var: `RATE_LIMIT_REPEATS`
        - Workspace secret: `prismeai_ratelimit_repeats`
      </Card>
    </CardGroup>

    <Warning>
      Runtime rate limits are per workspace per Runtime instance. In a cluster with N instances, effective limits are N times the configured values.
    </Warning>
  </Tab>
</Tabs>

## Rate Limit Headers

Response headers included with every API request:

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Maximum requests allowed in the current window |
| `X-RateLimit-Remaining` | Requests remaining in the current window |
| `X-RateLimit-Reset` | Window reset time (Unix epoch seconds) |
| `Retry-After` | Seconds to wait before retrying (only present when rate limited) |

## Configuration

<Tabs>
  <Tab title="Global (Environment Variables)">
    | Variable | Description | Default |
    |----------|-------------|---------|
    | `RATE_LIMIT_AUTOMATIONS` | Automations/second | 100 |
    | `RATE_LIMIT_EMITS` | Event emits/second | 30 |
    | `RATE_LIMIT_FETCHS` | HTTP fetches/second | 50 |
    | `RATE_LIMIT_REPEATS` | Repeat iterations/second | 1000 |
    | `RATE_LIMIT_AUTOMATIONS_BURST` | Automations burst limit | 400 |
    | `RATE_LIMIT_EMITS_BURST` | Event emits burst limit | 100 |
    | `RATE_LIMIT_FETCHS_BURST` | HTTP fetches burst limit | 200 |
    | `RATE_LIMIT_REPEATS_BURST` | Repeat iterations burst limit | 4000 |
    | `RATE_LIMIT_DISABLED` | Disable all rate limits | false |

    Setting any value to `0` disables that specific rate limit for all workspaces.
  </Tab>
  <Tab title="Per-Workspace (Secrets)">
    | Workspace Secret | Description | Default |
    |------------------|-------------|---------|
    | `prismeai_ratelimit_automations` | Automations/second | 100 |
    | `prismeai_ratelimit_emits` | Event emits/second | 30 |
    | `prismeai_ratelimit_fetchs` | HTTP fetches/second | 50 |
    | `prismeai_ratelimit_repeats` | Repeat iterations/second | 1000 |
    | `prismeai_ratelimit_automations_burst` | Automations burst limit | 400 |
    | `prismeai_ratelimit_emits_burst` | Event emits burst limit | 100 |
    | `prismeai_ratelimit_fetchs_burst` | HTTP fetches burst limit | 200 |
    | `prismeai_ratelimit_repeats_burst` | Repeat iterations burst limit | 4000 |
    | `prismeai_ratelimit_disabled` | Disable rate limits for workspace | false |

    <Info>
      These workspace secrets are restricted to super admins.
    </Info>
  </Tab>
</Tabs>

## Distributing Load

Sequential execution on a single instance is limited to that instance's rate:

```yaml
# This runs on a single instance — hits single-instance limit
slug: sequential-processing
do:
  - repeat:
      on: 2000
      do:
        - callSomeOtherAutomation: {}
```

Distribute across instances using events:

```yaml
# Emit events that can be processed by any instance
slug: distributed-processing
do:
  - repeat:
      on: 2000
      do:
        - emit:
            event: triggerProcessing
            payload: {}

---
slug: processItem
when:
  events:
    - triggerProcessing
do: []
```

Use batching for controlled throughput:

```yaml
do:
  - repeat:
      on: '{{workQueue}}'
      batch:
        size: 3
        interval: 500
      do:
        - process:
            item: '{{item}}'
```

## Monitoring Throttling

Each automation execution generates a `runtime.automations.executed` event with throttling data:

```json
{
  "event": "runtime.automations.executed",
  "payload": {
    "automation": "my-automation",
    "workspace": "my-workspace",
    "duration": 1250,
    "throttled": 1000,
    "status": "success"
  }
}
```

If `throttled` is greater than zero, the automation was rate limited for that many milliseconds.
