The Prisme.ai API provides a comprehensive set of RESTful endpoints for programmatically interacting with the platform. It accepts JSON-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

## Key Concepts

<Accordion title="Microservices Architecture">
  The API is built on a microservices architecture. The api-gateway handles authentication, routing, and load balancing across backend services:

  - **API Gateway** — Authentication, session management, request routing
  - **Workspaces** — Workspace CRUD, DSUL storage, RBAC
  - **Runtime** — Automation execution, event processing
  - **Events** — Event storage, WebSocket streaming, Elasticsearch
  - **Pages** — End-user page rendering
</Accordion>

<Accordion title="Events System">
  Prisme.ai uses an event-driven architecture. The API allows you to:

  - Subscribe to events via WebSocket or webhooks
  - Emit events to trigger automations
  - Process events in real-time

  This enables complex workflows and integrations between workspaces and external systems.
</Accordion>

<Accordion title="Workspace-Based Products">
  Platform products (Agent Factory, AI Governance, AI Collection, etc.) are implemented as workspaces with REST API endpoints exposed via `when: endpoint:` automations. Each workspace defines its own API surface.

  See the [Architecture Overview](/architecture/overview) for details on the workspace-as-microservice pattern.
</Accordion>

## Making Your First Request

```bash
# Check system status (no auth required)
curl -X GET "https://api.your-domain.com/v2/"

# List workspaces (auth required)
curl -X GET "https://api.your-domain.com/v2/workspaces" \
     -H "Content-Type: application/json" \
     -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

<Warning>
  Replace `YOUR_ACCESS_TOKEN` with a valid token. See [Authentication](/api-reference/authentication) for details on obtaining tokens.
</Warning>
