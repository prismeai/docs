### ISO 27001

ISO 27001 is an internationally recognized standard for information security management. Our certification confirms that Prisme.ai:

- Systematically examines security risks
- Implements comprehensive security controls
- Adopts a management process to ensure controls meet security needs over time
- Maintains an information security management system (ISMS) that protects customer data

<Note>
  Prisme.ai's ISO 27001 certification covers our entire cloud infrastructure, application platform, and supporting business processes.
</Note>

## Security Frameworks and Standards

Prisme.ai aligns with leading security frameworks and standards:

### NIST Cybersecurity Framework

Our security program incorporates the five core functions of the NIST CSF:

<Steps>
  <Step title="Identify">
    Developing organizational understanding to manage cybersecurity risk
  </Step>
  <Step title="Protect">
    Implementing safeguards to ensure delivery of critical services
  </Step>
  <Step title="Detect">
    Implementing activities to identify cybersecurity events
  </Step>
  <Step title="Respond">
    Taking action regarding detected cybersecurity incidents
  </Step>
  <Step title="Recover">
    Maintaining resilience and restoring capabilities impaired by incidents
  </Step>
</Steps>

### Cloud Security Alliance (CSA)

Prisme.ai adheres to the Cloud Controls Matrix (CCM), providing:

- Transparency through the CSA STAR Registry
- Implementation of cloud-specific security controls
- Regular assessment against CSA best practices

### OWASP Security Practices

Our development processes incorporate OWASP security principles:

- Regular security testing for OWASP Top 10 vulnerabilities
- Secure coding practices
- Security requirements integrated into development lifecycle
- Static and dynamic application security testing

## Compliance FAQs

<AccordionGroup>
  <Accordion title="How often are security assessments conducted?">
    Prisme.ai conducts various security assessments on different schedules:

    - Automated vulnerability scanning: Daily
    - External penetration testing: Quarterly
    - Comprehensive security assessment: Annually
    - Third-party audits (ISO 27001, SOC 2): Annually
    - Code security reviews: With every major release
  </Accordion>
  <Accordion title="How does Prisme.ai handle data residency requirements?">
    Prisme.ai offers several options for customers with data residency requirements:

    - Regional data centers in North America, Europe, and Asia-Pacific
    - Data localization controls to restrict where data is stored and processed
    - Self-hosted options for complete control over data location

    Our platform architecture allows for data segregation by region, ensuring that data stays within designated geographical boundaries.
  </Accordion>
  <Accordion title="What security measures are in place for AI models?">
    Prisme.ai implements multiple security measures for AI components:

    - Data isolation between customers when training or fine-tuning models
    - Encryption of training data and model parameters
    - Strict access controls for model development and deployment
    - Regular evaluation for bias, security vulnerabilities, and privacy implications
    - Prompt injection and other AI-specific attack vector mitigation

    For customers using third-party AI services, we provide clear documentation on data handling practices and security considerations.
  </Accordion>
  <Accordion title="How does Prisme.ai handle security incidents?">
    Our incident response process follows industry best practices:

    1. Detection through monitoring and alerts
    2. Classification of incident severity
    3. Containment to limit potential damage
    4. Investigation to determine cause and impact
    5. Remediation of affected systems
    6. Notification to affected customers as required by contracts and regulations
    7. Post-incident analysis and improvement

    Our incident response team is available 24/7 to address security events.
  </Accordion>
</AccordionGroup>
