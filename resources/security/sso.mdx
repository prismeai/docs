## Supported Identity Providers

| Provider | Protocol | Status |
|----------|----------|--------|
| Microsoft Entra ID (Azure AD) | OIDC / SAML | Fully supported |
| Okta | OIDC / SAML | Fully supported |
| Google Workspace | OIDC | Fully supported |
| Auth0 | OIDC | Fully supported |
| OneLogin | SAML | Fully supported |
| PingIdentity | SAML | Fully supported |
| Keycloak | OIDC | Fully supported |
| ADFS | SAML | Fully supported |

## Protocols

<Tabs>
  <Tab title="OIDC">
    OpenID Connect (built on OAuth 2.0):
    - Authorization Code flow with PKCE
    - RS256 signing algorithm
    - Automatic key rotation
    - Scopes: `openid`, `email`, `profile`
  </Tab>
  <Tab title="SAML 2.0">
    Security Assertion Markup Language:
    - HTTP-POST binding
    - SP-initiated login
    - Signed assertions
    - Attribute mapping for user profile fields
  </Tab>
</Tabs>

## Provisioning

### Just-In-Time (JIT) Provisioning

Users are automatically created on first login via SSO. Attributes (name, email, groups) are mapped from the IdP.

### SCIM Provisioning

Automated user lifecycle management:

| Provider | SCIM Support |
|----------|-------------|
| Microsoft Entra ID | Supported |
| Okta | Supported |
| OneLogin | Supported |

SCIM enables:
- Automatic user creation and deactivation
- Group synchronization
- Profile attribute updates

### Group-Based Role Assignment

Map IdP groups to Prisme.ai roles:

```yaml
authorizations:
  roles:
    admin:
      auth:
        MySSO:
          groups:
            - platform-admins
    viewer:
      auth:
        MySSO:
          groups:
            - platform-users
```

## MFA Support

MFA is enforced at the Identity Provider level. When configured in your IdP, all Prisme.ai SSO logins require MFA.
