## Agent & Conversation Issues

<Accordion title="Agent not responding">
  1. Check LLM Gateway connectivity: `POST /v2/workspaces/slug:llm-gateway/webhooks/tests/run-all`
  2. Verify the agent's model exists in the LLM Gateway model catalog
  3. Check for rate limiting in `runtime.automations.executed` events (look for `throttled > 0`)
  4. Review agent configuration — ensure `model` and `instructions` are set
</Accordion>

<Accordion title="Streaming not working">
  1. Verify WebSocket connectivity to the Events service
  2. Check that the SSE endpoint is accessible (no proxy buffering)
  3. Ensure `Accept: text/event-stream` header is set
  4. Check nginx/ingress configuration for proxy buffering settings
</Accordion>

<Accordion title="Tools not being called">
  1. Verify tool definitions are valid in the agent configuration
  2. Check that MCP endpoints are reachable from the Runtime
  3. Review guardrails — action approval guardrails may be blocking tool calls
  4. Check agent profile — `simple` profile doesn't support tools
</Accordion>

## Authentication Issues

<Accordion title="401 Unauthorized">
  1. Check token expiration — default JWT lifetime is 30 days
  2. Verify the token format in the Authorization header: `Bearer <token>`
  3. For API keys, use `x-prismeai-api-key` header instead
  4. Check if the JWT signing key has been rotated
</Accordion>

<Accordion title="SSO login fails">
  1. Verify redirect URI matches exactly: `https://API_URL/v2/login/callback`
  2. Check IdP certificate validity for SAML
  3. Monitor `gateway.login.started` and `gateway.login.succeeded` events
  4. Verify `ENABLED_AUTH_PROVIDERS` env var matches your config
</Accordion>

## Deployment Issues

<Accordion title="Pods in CrashLoopBackOff">
  ```bash
  kubectl logs <pod-name> -n prisme --previous
  ```
  Common causes:
  - Incorrect database connection strings
  - Missing Redis connection
  - Insufficient memory (increase limits)
  - Missing configuration files
</Accordion>

<Accordion title="Platform frontend shows blank page">
  1. Check `/env-config.js` returns valid JavaScript
  2. Verify `API_URL` environment variable is set correctly
  3. Check browser console for CORS errors
  4. Ensure the API Gateway is reachable from the frontend
</Accordion>

<Accordion title="Workspace push fails">
  1. Verify API connectivity and authentication
  2. Check workspace slug doesn't conflict with existing workspaces
  3. Validate YAML syntax in workspace files
  4. Check for circular dependencies in imports
</Accordion>

## Performance Issues

<Accordion title="Slow API responses">
  1. Check Elasticsearch cluster health: `GET /_cluster/health`
  2. Monitor Redis memory: `redis-cli INFO memory`
  3. Check Runtime throttling in `runtime.automations.executed` events
  4. Review MongoDB/PostgreSQL slow query logs
  5. Consider scaling Runtime replicas (rate limits are per-instance)
</Accordion>

<Accordion title="High memory usage">
  1. Check for memory leaks in Elasticsearch (review heap usage)
  2. Monitor Redis memory usage and eviction policy
  3. Review Kubernetes resource limits vs. actual usage
  4. Consider implementing event retention policies
</Accordion>

## Getting Help

If you can't resolve an issue:

1. Collect the `x-correlation-id` from the error response
2. Export relevant events using the Events API
3. Check the [community forum](https://prisme-ai.discourse.group)
4. Contact support at support@prisme.ai with correlation IDs and logs
