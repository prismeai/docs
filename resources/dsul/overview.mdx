DSUL (Declarative Stack Unified Language) is the open-source declarative language at the heart of Prisme.ai. It enables organizations to describe, build, and orchestrate AI-powered applications using YAML-based definitions rather than imperative code.

## Core Components

| Component | Description |
|-----------|-------------|
| **Pages** | Declarative UI containers defining views and user journeys |
| **Blocks** | Reusable UI components encapsulating specific functionality |
| **Automations** | Backend processes orchestrating logic, workflows, and AI operations |
| **Integrations** | Connectors to external systems, databases, and services |
| **Workspaces** | Project environments encapsulating all resources and configurations |
| **RBAC** | Security model for fine-grained access control |

## How DSUL Powers the Platform

Every workspace in the Prisme.ai platform — from Agent Factory to AI Governance — is built entirely in DSUL:

- **REST APIs** are `when: endpoint:` automations
- **Data models** are Collection App imports (`imports/*.yml`)
- **Scheduled jobs** are `when: cron:` automations
- **Event handlers** are `when: events:` automations
- **Security rules** are `security.yml` RBAC definitions
- **Inter-service calls** use `fetch` (cross-workspace, identified by `run.sourceWorkspaceId`)

See [Workspace Development](/development/workspace-development) for a hands-on guide to writing DSUL automations.

## Example: REST API in DSUL

```yaml
slug: v1-items-list
name: List Items
when:
  endpoint: v1/items
  method: GET
do:
  - Storage.v1.search:
      workspace_id: '{{workspace_id}}'
      collection: items
      query: '{{query.search}}'
      limit: '{{query.limit | default: 20}}'
      output: results

  - output:
      items: '{{results.items}}'
      total: '{{results.total}}'
```

## Example: Event-Driven Automation

```yaml
slug: on-item-created
name: Process New Item
when:
  events:
    - items.created
do:
  - LLM-Gateway.v1.chat-completions:
      model: gpt-4o-mini
      messages:
        - role: system
          content: Classify this item
        - role: user
          content: '{{event.payload.description}}'
      output: classification

  - Storage.v1.update:
      id: '{{event.payload.id}}'
      data:
        category: '{{classification.choices[0].message.content}}'
```
