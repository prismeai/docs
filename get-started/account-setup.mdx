# Account Setup and Configuration

Setting up your Prisme.ai account correctly ensures you can make the most of the platform's capabilities. This guide walks you through the essential configuration steps and options.

## User Account Setup

<Steps>
  <Step title="Complete your profile">
    After your initial login, navigate to your profile settings by clicking on your avatar in the top-right corner and selecting "Profile."
    
    <Frame>
      <img src="/images/profile-menu.png" alt="Profile menu" />
    </Frame>
    
    Fill out your profile information, including:
    - Display name
    - Job title
    - Department
    - Profile picture (optional)
    
    This information helps other users identify you when collaborating on the platform.
  </Step>
  
  <Step title="Set your preferences">
    Configure your user preferences to customize your Prisme.ai experience:
    
    <Frame>
      <img src="/images/preferences.png" alt="User preferences" />
    </Frame>
    
    Key settings include:
    - UI language
    - Theme (light/dark/system)
    - Default LLM model (where applicable)
    - Notification preferences
    - Time zone
  </Step>
  
  <Step title="Configure authentication methods">
    Enhance your account security by setting up additional authentication methods:
    
    <Frame>
      <img src="/images/auth-methods.png" alt="Authentication methods" />
    </Frame>
    
    Options may include:
    - Password change
    - Two-factor authentication (2FA)
    - SSO configuration (if supported by your organization)
    
    <Note>
      For enterprise deployments with SSO, some of these options may be managed by your IT department.
    </Note>
  </Step>
  
  <Step title="Set API access (if needed)">
    If you'll be using the Prisme.ai API, you can generate and manage your API keys:
    
    <Frame>
      <img src="/images/api-keys.png" alt="API keys management" />
    </Frame>
    
    Remember to:
    - Keep your API keys secure
    - Use descriptive names for different keys
    - Rotate keys periodically for enhanced security
    - Set appropriate scopes and permissions
  </Step>
</Steps>

## Workspace Configuration

If you have admin privileges, you may also need to configure workspace settings for your team or organization:

<Steps>
  <Step title="Access workspace settings">
    Navigate to the admin panel by clicking on your avatar and selecting "Admin Settings" or "Workspace Configuration" (the exact name may vary depending on your deployment).
    
    <Frame>
      <img src="/images/admin-panel.png" alt="Admin panel" />
    </Frame>
  </Step>
  
  <Step title="Manage users and groups">
    Add users to your workspace and organize them into groups for easier permission management:
    
    <Frame>
      <img src="/images/user-management.png" alt="User management" />
    </Frame>
    
    You can:
    - Invite new users
    - Create and manage groups
    - Assign users to groups
    - Set group permissions
    - Configure SSO integration
  </Step>
  
  <Step title="Configure LLM settings">
    Set up the language models available in your workspace:
    
    <Frame>
      <img src="/images/llm-settings.png" alt="LLM settings" />
    </Frame>
    
    Options include:
    - Selecting available models
    - Setting API keys for external LLM providers
    - Configuring default models for different products
    - Setting usage quotas and limits
  </Step>
  
  <Step title="Set up integrations">
    Configure integrations with other systems your organization uses:
    
    <Frame>
      <img src="/images/integrations.png" alt="Integrations" />
    </Frame>
    
    Common integrations include:
    - Authentication systems (SSO, LDAP, etc.)
    - Storage systems (S3, GCS, etc.)
    - Monitoring and logging tools
    - Enterprise systems (CRM, ERP, ITSM, etc.)
  </Step>
</Steps>

## Role-Specific Settings

Different user roles may need specific configurations:

<Tabs>
  <Tab title="End Users">
    As an end user, focus on:
    - Setting up your profile and preferences
    - Configuring notification settings
    - Personalizing your AI interaction settings
    
    You generally won't need to worry about advanced settings like API keys or workspace configuration.
  </Tab>
  
  <Tab title="Power Users">
    As a power user or agent creator, you should also:
    - Set up API keys if you'll be using them
    - Configure default settings for agent creation
    - Set up connections to knowledge bases you manage
  </Tab>
  
  <Tab title="Developers">
    As a developer, pay attention to:
    - API key configuration with appropriate scopes
    - Webhook settings for integrations
    - Development environment connections
    - CI/CD integration settings (if applicable)
  </Tab>
  
  <Tab title="Administrators">
    As an administrator, you'll need to configure all aspects of the workspace:
    - User management and permissions
    - LLM provider settings and API keys
    - Security policies and compliance settings
    - Usage quotas and monitoring
    - Backup and disaster recovery options
  </Tab>
</Tabs>
