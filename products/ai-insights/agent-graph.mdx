---
title: "Agent Graph"
description: "Visual relationship mapping between agents, tools, knowledge bases, and users"
---

The Agent Graph provides a visual representation of how agents, tools, knowledge bases, and data sources are interconnected. It helps understand the platform topology and identify optimization opportunities.

## Endpoints

### Agent Relationships

```
GET v1/graph/agents
```

Returns all agent nodes with their connections:

```json
{
  "nodes": [
    {
      "id": "agent_abc",
      "type": "agent",
      "name": "Research Assistant",
      "profile": "agent_full",
      "metrics": {
        "conversations": 150,
        "avg_rating": 4.2,
        "tools_used": 5
      }
    },
    {
      "id": "vs_xyz",
      "type": "knowledge_base",
      "name": "Company Docs"
    },
    {
      "id": "tool_bing",
      "type": "tool",
      "name": "bing_search"
    }
  ],
  "edges": [
    {
      "source": "agent_abc",
      "target": "vs_xyz",
      "type": "uses_knowledge",
      "weight": 0.8
    },
    {
      "source": "agent_abc",
      "target": "tool_bing",
      "type": "uses_tool",
      "weight": 0.3
    }
  ]
}
```

### Agent Clusters

```
GET v1/graph/agents/clusters
```

Groups agents by similarity (shared tools, knowledge bases, or usage patterns):

```json
{
  "clusters": [
    {
      "id": "cluster_1",
      "name": "Research Agents",
      "agents": ["agent_abc", "agent_def"],
      "shared_tools": ["bing_search", "knowledge_base"],
      "similarity_score": 0.85
    }
  ]
}
```

### Similar Agents

```
GET v1/graph/similar/:agent_id
```

Find agents similar to a given agent based on configuration, tools, and usage patterns. Useful for deduplication and consolidation.

### Tool Relationships

```
GET v1/graph/tools
```

Returns tool usage across agents:

```json
{
  "tools": [
    {
      "id": "knowledge_base",
      "type": "file_search",
      "used_by": ["agent_abc", "agent_def", "agent_ghi"],
      "total_calls": 5200,
      "avg_latency_ms": 350
    }
  ]
}
```

### Full Relationship Map

```
GET v1/graph/relationships
```

Returns the complete relationship graph including agents, tools, knowledge bases, users, and groups. This is the data source for the graph visualization in the AI Insights frontend.

## Frontend Visualization

The AI Insights frontend uses `@xyflow/react` (React Flow) to render the agent graph as an interactive node-and-edge diagram. Users can:

- Zoom and pan
- Click nodes for details
- Filter by agent profile, tool type, or cluster
- View metrics overlays (usage, ratings, cost)

## Data Storage

Graph data is stored in two collections:

| Collection | Purpose |
|-----------|---------|
| `agent_graph_nodes` | Node records (agent, tool, knowledge base, group) |
| `agent_graph_edges` | Edge records with type, weight, and metadata |

The graph is rebuilt periodically from Agent Factory data and usage analytics.
