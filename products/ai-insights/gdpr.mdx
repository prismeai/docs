AI Insights includes GDPR compliance tools for handling user data rights — the right to erasure, the right to data portability, and data anonymization.

## Data Deletion

### Request Deletion

```
POST v1/gdpr/users/:user_id/deletion
```

Creates a deletion request record in `gdpr_requests` with status `pending`. No request body is needed — the user ID comes from the path parameter and authentication context.

### Deletion Processing

Pending deletion requests are processed nightly (`0 3 * * *`) by `gdpr/process-pending-requests`:

1. Fetch requests approaching their 30-day GDPR deadline
2. For each request, call `gdpr/process-deletion`:
   - Delete insights records
   - Delete analysis queue items
   - Delete aggregated statistics
3. Update request status to `completed`
4. Log the deletion for audit

### Deletion Status

Deletion requests can be tracked through the `gdpr_requests` collection. Statuses:

| Status | Description |
|--------|-------------|
| `pending` | Request created, waiting for nightly processing |
| `processing` | Currently being executed |
| `completed` | All data successfully deleted |
| `partial` | Some deletions succeeded, others failed (retried next night) |
| `failed` | Deletion failed (requires manual intervention) |

## Data Export

```
GET v1/gdpr/users/:user_id/export
```

Exports all user data. Returns:

```json
{
  "download_url": "https://...",
  "expires_at": "2026-03-02T10:00:00Z",
  "record_counts": {
    "insights": 42,
    "feedback": 15
  }
}
```

## Text Anonymization

The `_anonymize-text` automation supports three anonymization modes:

| Mode | Description |
|------|-------------|
| `hash` | Replace PII with one-way hashes (preserves uniqueness for analytics) |
| `redact` | Replace PII with type placeholders (`[EMAIL]`, `[PHONE]`, etc.) |
| `pseudonymize` | Replace PII with consistent fake values (reversible with key) |

Anonymization is used when retaining analytics data after a deletion request or during data retention policy enforcement.

## Retention Policies

Retention policies define how long data is kept before automatic purging:

```json
{
  "retention_days": 90,
  "auto_anonymize_after_days": 365,
  "applies_to": ["insights", "feedbacks"],
  "is_default": true,
  "enabled": true
}
```

Policies are stored in the `retention_policies` collection and enforced by the nightly purge job (`0 4 * * *`).

The `_resolve-retention-policy` automation determines the applicable policy for each data type, considering organization-level overrides.

## Data Expiry

The nightly purge job (`gdpr/purge-expired-data`) at `0 4 * * *`:

1. Reads retention policies for the organization
2. Identifies records past their TTL
3. Anonymizes records past the `anonymize_after_days` threshold
4. Deletes records past the `ttl_days` threshold
5. Logs all operations for audit
