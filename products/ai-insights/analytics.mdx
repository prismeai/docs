AI Insights provides multiple analytics endpoints covering different aspects of platform usage.

## Summary

```
GET v1/analytics/summary
```

High-level platform analytics:
- Total conversations, messages, and active users
- Average response time and satisfaction scores
- LLM token usage and cost
- Trend data (vs. previous period)

## Agent Analytics

```
GET v1/analytics/agents
```

Per-agent performance metrics:

| Metric | Description |
|--------|-------------|
| Conversation count | Total conversations per agent |
| Message count | Total messages sent/received |
| Average rating | User satisfaction score |
| Resolution rate | Percentage of resolved conversations |
| Average response time | Time to first response |
| Tool usage | Frequency of tool calls per type |
| Token consumption | Input/output tokens per conversation |

Supports filtering by:
- `agent_id` — Specific agent
- `period` — Time range (`24h`, `7d`, `30d`, `90d`)
- `granularity` — Data point interval (`1h`, `1d`, `1w`)

## Feedback Analytics

```
GET v1/analytics/feedback
```

Aggregated user feedback:
- Rating distribution (1-5 stars)
- Feedback trends over time
- Top feedback themes (from text analysis)
- Per-agent feedback breakdown

### Submit Feedback

```
POST v1/feedback
```

```json
{
  "agent_id": "agent_abc",
  "conversation_id": "conv_xyz",
  "rating": 4,
  "comment": "Very helpful, but could be faster",
  "tags": ["helpful", "slow"]
}
```

### List Feedback

```
GET v1/feedback
```

Paginated feedback list with filtering by agent, rating, and date range.

## Queue Statistics

```
GET v1/analytics/queue-stats
```

Analysis queue health:
- Pending analyses
- Processing rate
- Average processing time
- Queue depth over time
- Failed analyses and retry count

## Capacity Analytics

```
GET v1/analytics/capacity
```

Platform capacity planning data:
- Current utilization vs. limits
- Projected usage trends
- Bottleneck identification
- Scaling recommendations

## Adoption Analytics

```
GET v1/analytics/adoption
```

User adoption metrics:
- New users over time
- Active users (DAU, WAU, MAU)
- Feature adoption rates
- User retention curves
- First-time-to-regular-use conversion

## Organization-Level Aggregation

Organization analytics are aggregated hourly by a scheduled job (`10 * * * *`) and stored in the `org_analytics` collection.

```
POST v1/admin/refresh-org-analytics
```

Force-refresh organization analytics (admin only). Useful after large data imports or migrations.

## Insights Status

```
GET v1/insights/status
```

Returns the analysis status for the current organization:
- Total conversations analyzed
- Pending analyses
- Last analysis timestamp
- Analysis coverage percentage
