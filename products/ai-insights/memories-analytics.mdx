AI Insights tracks how agents use [long-term memory](/tools/memories) and provides analytics for understanding memory patterns, identifying gaps, and detecting anomalies.

## Memory Statistics

### Hourly Stats

```
GET v1/analytics/memories
```

Returns memory usage statistics:

| Metric | Description |
|--------|-------------|
| Total memories | Count of all stored memories |
| Memories per user | Average memories per user |
| Memories per agent | Average memories per agent |
| Memory types distribution | Breakdown by type (fact, preference, relationship, instruction) |
| Recall hit rate | Percentage of recalls that returned relevant memories |
| Write/read ratio | Ratio of remember vs. recall operations |

Supports filtering by:
- `agent_id` — Specific agent
- `period` — Time range (`24h`, `7d`, `30d`)
- `granularity` — Data point interval

### Trend Analysis

```
GET v1/analytics/memories/trends
```

Memory growth and usage trends:
- Memory creation rate over time
- Recall frequency trends
- Memory type distribution changes
- Per-agent memory growth

## Topic Clustering

```
GET v1/analytics/memories/topics
```

Groups memories into topic clusters using LLM analysis:

```json
{
  "topics": [
    {
      "name": "Product Preferences",
      "memory_count": 45,
      "top_keywords": ["product", "feature", "preference", "like"],
      "agents": ["agent_abc", "agent_def"],
      "growth_rate": 0.15
    },
    {
      "name": "Technical Issues",
      "memory_count": 32,
      "top_keywords": ["error", "bug", "fix", "troubleshoot"],
      "agents": ["agent_support"],
      "growth_rate": -0.05
    }
  ]
}
```

Topic clustering runs hourly (`0 * * * *`) and stores results in `memory_topics`.

## Knowledge Gaps

```
GET v1/analytics/memories/gaps
```

Identifies areas where memories are sparse or missing:
- Topics frequently asked about but with few stored memories
- Agents with low memory coverage
- Users with high recall miss rates
- Suggested topics to improve memory coverage

## Anomaly Detection

A scheduled job (`*/15 * * * *`) monitors memory patterns and flags anomalies:

- **Sudden spikes** in memory creation (potential data quality issue)
- **Unusually high recall miss rates** (embedding quality degradation)
- **Memory deletion patterns** (potential privacy concern)
- **Cross-agent memory leakage** (isolation violation)

Detected anomalies are stored in `memory_alerts` and surfaced in the Insights dashboard.

## Data Aggregation

Memory statistics are aggregated at two levels:

| Schedule | Job | Output |
|----------|-----|--------|
| `20 * * * *` | Hourly aggregation | `memory_stats_hourly` |
| `30 2 * * *` | Daily rollup | `memory_stats_daily` |

Hourly stats power real-time dashboards. Daily stats power trend analysis and reporting.
