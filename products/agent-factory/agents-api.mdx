## Base URL

```
{API_URL}/workspaces/slug:agent-factory/webhooks/v1/agents
```

## Endpoints

### List Agents

```
GET v1/agents
```

Query parameters:

| Param | Type | Description |
|-------|------|-------------|
| `limit` | integer | Results per page (default: 20) |
| `offset` | integer | Pagination offset |
| `profile` | string | Filter by profile (`simple`, `workflow`, `agent_light`, `agent_full`, `orchestrator`) |
| `search` | string | Search by name or description |

Returns a paginated list of agents the authenticated user has access to.

### Create Agent

```
POST v1/agents
```

```json
{
  "name": "Research Assistant",
  "description": "Helps with research tasks",
  "profile": "agent_light",
  "model": "gpt-4o",
  "system_prompt": "You are a research assistant...",
  "tools": [
    {
      "type": "system",
      "name": "knowledge_search"
    },
    {
      "type": "custom",
      "name": "my-mcp-tool",
      "config": { ... }
    }
  ],
  "guardrails": [
    {
      "type": "input",
      "name": "prompt_injection_detection",
      "config": { "sensitivity": "high" }
    }
  ]

```

The profile determines which features and limits are available. See [Agent Profiles](/products/agent-factory/overview#agent-profiles).

### Get Agent

```
GET v1/agents/:agent_id
```

Returns the full agent configuration including tools, guardrails, and metadata.

### Update Agent

```
PATCH v1/agents/:agent_id
```

Accepts the same body as creation. Emits `agents.updated` event for cache invalidation.

### Delete Agent

```
DELETE v1/agents/:agent_id
```

Deletes the agent and all associated data (conversations, tasks, bindings). Emits `agents.deleted` event.

## Access Control

Agents support fine-grained access control through bindings — linking agents to users or groups.

### Get Access

```
GET v1/agents/:agent_id/access
```

Returns the list of principals (users, groups) with access to the agent.

### Remove Access

```
DELETE v1/agents/:agent_id/access/:principal_type/:principal_id
```

Removes access for a specific user, group, or org. `principal_type` must be one of `user`, `group`, or `org`.

When a group is deleted (event `iam.group.deleted`), all agent bindings referencing that group are automatically cleaned up.

### Access Requests

Users can request access to agents they don't have permission to use:

```
POST v1/agents/:agent_id/access/request        # Request access (user-facing)
GET  v1/agents/:agent_id/access/requests        # List pending requests (owner)
POST v1/agents/:agent_id/access/requests/:id    # Approve or reject a request
```

## Analytics

### Get Analytics

```
GET v1/agents/:agent_id/analytics
```

Returns usage analytics for an agent over a given time window — users, conversations, messages, LLM usage, ratings, cost, latency, and more.

**Query parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `period` | string | `30d` | Time window: `1d`, `7d`, `30d`, or `90d` |
| `granularity` | string | *(auto)* | Series resolution: `hourly` or `daily`. If omitted, defaults to `hourly` for 1d/7d and `daily` for 30d/90d |

**Response:**

```json
{
  "agent_id": "agent_xxx",
  "agent_name": "My Agent",
  "period": {
    "type": "30d",
    "start": "2026-03-22T07:00:00.000Z",
    "end": "2026-04-21T07:00:00.000Z",
    "granularity": "daily"
  },
  "summary": {
    "users": { "active": 42, "new": 0, "returning": 0, "churned": 0 },
    "engagement": {
      "conversations": 120,
      "messages": 350,
      "llm_calls": 1200,
      "llm_calls_per_message": 3.43,
      "avg_messages_per_conversation": 2.9,
      "avg_session_duration_ms": 0
    },
    "quality": {
      "ratings_avg": 4.5,
      "ratings_count": 20,
      "ratings_sum": 90,
      "ratings_distribution": { "1": 0, "2": 1, "3": 2, "4": 5, "5": 12 },
      "resolution_rate": 0
    },
    "performance": { "p50_ms": 1200, "p95_ms": 4500, "error_count": 2, "error_rate": 0.006 },
    "cost": {
      "tokens_input": 500000,
      "tokens_output": 25000,
      "tokens_total": 525000,
      "cost": 1.25,
      "carbon_kgco2": 0.85
    },
    "models": {
      "claude-sonnet-4-5-20250929": { "calls": 200, "tokens": 400000, "cost": 1.1 },
      "gpt-4o": { "calls": 50, "tokens": 125000, "cost": 0.15 }
    },
    "tools": {
      "tool_calls": 80,
      "bing_search": { "calls": 40 },
      "knowledge_base": { "calls": 30 }
    }
  },
  "series": [
    { "date": "2026-03-22T00:00:00.000Z", "users": 5, "conversations": 8, "messages": 20, "rating": 4.5, "cost": 0.04, "status": "complete" },
    { "date": "2026-03-23T00:00:00.000Z", "users": 3, "conversations": 5, "messages": 12, "rating": null, "cost": 0.03, "status": "complete" }
  ],
  "meta": {
    "generated_at": "2026-04-21T07:00:00.000Z",
    "data_points": 30,
    "includes_current_interval": true,
    "summary_cached": true
  }
}
```

**Granularity behavior:**

- `1d` / `7d` → hourly points (up to 24 or 168 data points)
- `30d` / `90d` → daily points (up to 30 or 90 data points)
- Can be overridden with `?granularity=hourly` or `?granularity=daily`

**Performance & caching:**

- **Summary caching**: The summary is cached for 1 hour in the `agent_metrics` collection (`period_type: summary`). Subsequent requests within the TTL skip the expensive Elasticsearch aggregation queries.
- **Series self-healing**: On each request, the endpoint checks that all expected intervals have been pre-aggregated. If gaps are found (e.g., the cron job missed a run), it triggers on-demand aggregation before returning.
- **Current interval**: The current (in-progress) hour or day is refreshed if its cached value is older than 15 minutes, ensuring near-real-time data without constant re-computation.

**Series `status` field:**

Each data point includes a `status` field:
- `complete` — The interval has ended and the metrics are final
- `incomplete` — The interval is still in progress (current hour/day); values will increase

### Refresh Metrics

```
POST v1/agents/:agent_id/refresh-metrics
```

Force a re-aggregation of metrics for the agent. Useful after bulk imports or when metrics appear stale.

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `hours_back` | integer | `1` | How many hours to re-aggregate (max: 168 = 1 week) |

This endpoint:
1. Invalidates all cached summaries for the agent
2. Re-runs the bulk aggregation pipeline for the requested time window

**Response:**

```json
{
  "status": "refreshed",
  "agent_id": "agent_xxx",
  "hours_back": 1,
  "entries_written": 3
}
```

### Background Jobs

| Job | Schedule | Description |
|-----|----------|-------------|
| `jobs/aggregate-agent-metrics` | `0 * * * *` (hourly) | Aggregates hourly metrics for all active agents from Elasticsearch events |
| `jobs/cleanup-agent-metrics` | `0 3 * * *` (daily) | Deletes metric rows older than 30 days |
| `jobs/refresh-agent-counters` | `*/15 * * * *` (every 15 min) | Refreshes per-agent conversation and message counters |

### Event Sources

Analytics are computed from two Elasticsearch event streams:

| Event | Source Workspace | Data |
|-------|-----------------|------|
| `agents.conversations.created` | agent-factory | User ID, conversation ID → unique users, conversation count |
| `analytics.agent.rated` | agent-factory | Rating value → quality metrics |
| `analytics.llm.completion` | llm-gateway | Model, tokens, cost, duration, tool calls → engagement, performance, cost, model/tool breakdowns |

## A2A Protocol

Agent Factory implements the [Agent-to-Agent (A2A) Protocol](https://google.github.io/A2A/) for interoperability:

### Agent Card

```
GET v1/agents/:agent_id/.well-known/agent.json
GET .well-known/agent.json
```

Returns the A2A agent card — a standardized description of the agent's capabilities, supported content types, and endpoint URL.

### Extended Agent Card

```
GET v1/agents/:agent_id/extendedAgentCard
```

Returns the full agent card with additional Prisme.ai-specific metadata.

### Discard Draft

```
POST v1/agents/:agent_id/discard-draft
```

Discards unpublished changes to an agent, reverting to the last published version.

### A2A Endpoint

```
POST v1/agents/:agent_id/a2a
```

The standard A2A protocol endpoint. Accepts A2A-formatted requests for the specified agent.

## Retention

Configure data retention policies per agent or per organization:

```
GET/PUT/DELETE v1/agents/:agent_id/retention    # Per-agent retention policy
GET/PUT        v1/orgs/:orgSlug/retention       # Org-level retention defaults
```

Retention policies control how long conversations, tasks, artifacts, and uploaded documents are kept before automatic purging. A daily job (`0 4 * * *`) enforces these policies.

## Authentication

Agent Factory supports all three authentication modes:

| Mode | Use Case |
|------|----------|
| **User session** | Frontend (SecureChat, Agent Factory UI) |
| **API key (`iak_*`)** | External integrations, programmatic access |
| **Cross-workspace** | Other workspaces (AI Insights, Evaluations) |

API key validation results are cached for 60 seconds. User session lookups are also cached for 60 seconds per user.
