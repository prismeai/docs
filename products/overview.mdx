## Product Integration

While each Prisme.ai product serves a specific purpose, they are designed to work together seamlessly. Here's how they integrate:

### Common Integration Patterns

1. **Knowledge-Powered Agents**
   - Create document stores in Knowledges
   - Build and configure agents in Agent Creator
   - Connect knowledge bases to agents as capabilities
   - Access through Chat
2. **Data-Driven Solutions**
   - Connect to structured data through Builder imports and workflows
   - Build custom interfaces with Builder pages
   - Track performance with Insights
3. **Enterprise Deployment**
   - Manage access with Governe
   - Create and share agents through Agent Creator
   - Monitor usage with Insights

### Step-by-Step: Building a Knowledge-Powered Agent

This walkthrough shows how Knowledges, Agent Creator, and Chat work together:

<Steps>
  <Step title="Create a knowledge base (Knowledges)">
    1. Go to **Create > Knowledges**
    2. Click **Create Knowledge Base**
    3. Upload your documents or connect a data source
    4. Wait for documents to be processed and indexed
  </Step>
  <Step title="Create an agent (Agent Creator)">
    1. Go to **Create > Agent Creator**
    2. Click **Create Agent**
    3. Choose a profile (Light Agent works well for RAG)
    4. Write instructions explaining the agent's purpose
  </Step>
  <Step title="Connect knowledge to agent (Agent Creator)">
    1. Open your agent's **Capabilities** section
    2. Click **Add Capability**
    3. Select your knowledge base from the list
    4. Save the capability
  </Step>
  <Step title="Test and refine (Agent Creator)">
    1. Open the **Playground**
    2. Ask questions that should use the knowledge base
    3. Verify the agent retrieves relevant information
    4. Adjust instructions if needed
  </Step>
  <Step title="Share and use (Chat)">
    1. Publish the agent in **Settings > Sharing**
    2. Users find it in the **Store**
    3. They start conversations from **Chat**
  </Step>
</Steps>

### The Three-Product Flow

```
Knowledges          Agent Creator         Chat
     │                      │                     │
     │  Documents &         │  Agent config       │  End users
     │  embeddings          │  & capabilities     │  chat here
     │                      │                     │
     └──────────────────────┴─────────────────────┘
              Knowledge powers agent responses
```

- **Knowledges** stores and indexes your documents
- **Agent Creator** configures how the agent uses that knowledge
- **Chat** is where users interact with the finished agent
