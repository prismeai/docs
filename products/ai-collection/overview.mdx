AI Collection provides structured data storage for AI agents. It allows agents to create collections (tables), insert records, and query data using a familiar CRUD interface. It implements the [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) so agents can use it as a tool.

**Workspace:** `ai-collection-v3`

## Two Access Modes

AI Collection supports two ways to store data:

### 1. Agent-Scoped (Default)

Data is automatically isolated per agent and tool. When an agent stores data without specifying a collection, it goes into a default shared collection with automatic scoping:

```json
{
  "tool": "data_insert",
  "arguments": {
    "records": [{ "name": "Acme Corp", "revenue": 1500000 }]
  }
}
```

The `agent_id` and `tool_id` are injected automatically from the execution context. Queries are automatically filtered to the calling agent's data.

### 2. Named Collections

Named collections are explicitly created and can be shared across agents:

```json
{
  "tool": "data_create_collection",
  "arguments": {
    "name": "Companies",
    "properties": {
      "name": { "type": "text" },
      "revenue": { "type": "integer" },
      "industry": { "type": "text" }
    }
  }
}
```

Named collections have their own access control — the owner and authorized agents can read/write.

## MCP Integration

AI Collection exposes a full MCP JSON-RPC 2.0 endpoint for agent tool use:

```
POST /v2/workspaces/slug:ai-collection-v3/webhooks/ai-collection/mcp
```

**Protocol:** `2025-06-18`
**Capabilities:** `{ tools: {} }`

### Available Tools

| Tool | Description |
|------|-------------|
| `data_create_collection` | Create a named collection with typed properties |
| `data_insert` | Insert one or more records |
| `data_query` | Query records with filters, sorting, pagination |
| `data_get` | Get a specific record by ID |
| `data_update` | Update records matching a filter |
| `data_upsert` | Insert or update records |
| `data_delete` | Delete records matching a filter |
| `data_count` | Count records matching a filter |
| `data_distinct` | Get distinct values for a field |
| `data_aggregate` | Run aggregation queries |

The tool schemas are defined in the workspace's `config.mcpTools` and served at `tools/list`.

## Authentication

Authentication is handled via the Prisme.ai user session. The `_auth` automation resolves the authenticated user from the session. For MCP calls, the context (`agent_id`, `tool_id`, `owner_id`) is passed as arguments by Agent Factory.

## Configuration

| Setting | Default | Description |
|---------|---------|-------------|
| `default_ttl_days` | 0 (no expiry) | Default TTL for records |
| `default_max_records` | 10,000 | Maximum records per collection |
| `track_access` | false | Track last access time on queried records |

See [API Reference](/products/ai-collection/api) for detailed endpoint documentation.
