API keys (`iak_*`) enable programmatic access to the Prisme.ai platform from external systems. They are scoped to an organization and carry specific permissions and resource scopes.

## Key Format

All API keys use the format `iak_{orgSlug}_{uuid}`:

```
iak_acme-corp_a1b2c3d4e5f6g7h8...
```

The embedded `orgSlug` allows the auth middleware to resolve the organization without a database lookup. Keys are stored **hashed** in the `iam_apikeys` collection. A truncated prefix is stored for identification. The plaintext key is returned only once at creation time.

## Endpoints

### List API Keys

```
GET v1/orgs/:org_slug/apikeys
```

Returns all API keys for the organization (without the secret values).

### Create API Key

```
POST v1/orgs/:org_slug/apikeys
```

```json
{
  "name": "Production Integration",
  "permissions": [
    "llm:completions:use",
    "agents:read",
    "agents:use",
    "memories:read",
    "memories:write"
  ],
  "scopes": [
    "models:gpt-4o",
    "models:gpt-4o-mini",
    "agents:agent_abc123"
  ],
  "expiresAt": "2026-12-31T23:59:59Z"
}
```

**Response** (key is only shown once):
```json
{
  "success": true,
  "apikey": {
    "id": "key_xyz",
    "key": "iak_acme-corp_a1b2c3d4e5f6g7h8...",
    "name": "Production Integration",
    "permissions": [...],
    "scopes": [...]
  }
}
```

### Get / Update API Key

```
GET v1/orgs/:org_slug/apikeys/:key_id
PUT v1/orgs/:org_slug/apikeys/:key_id
```

Update allows changing name, permissions, scopes, and expiration. The key value itself cannot be changed (use rotation instead).

### Revoke API Key

```
POST v1/orgs/:org_slug/apikeys/:key_id/revoke
```

Immediately invalidates the key. Revoked keys cannot be un-revoked.

### Rotate API Key

```
POST v1/orgs/:org_slug/apikeys/:key_id/rotate
```

Generates a new key value while preserving the key's permissions and scopes. The old key is immediately invalidated. Returns the new plaintext key.

## Permissions

API key permissions follow the same `product:resource:action` pattern as role permissions, with wildcard support. See [Roles > Permission Format](/products/ai-governance/organizations#permission-format) for the full permission catalog.

Common API key permissions:

| Permission | Description |
|-----------|-------------|
| `llm:completions:use` | Access LLM Gateway chat completions |
| `llm:embeddings:use` | Access LLM Gateway embeddings |
| `agent-factory:agents:read` | Read agent configurations |
| `agent-factory:agents:write` | Create and modify agents |
| `storage:vector_stores:read` | Read knowledge bases |
| `storage:files:read` | Read files |
| `orgs:members:manage` | Manage organization members |
| `orgs:apikeys:manage` | Manage API keys |
| `orgs:*` | Full org access (admin) |

Wildcard permissions are supported: `storage:*` grants all storage permissions.

## Scopes

Scopes restrict **which resources** a key can access, even if the permission allows the action. They use the same `product:resource:id` format as role scopes:

```json
{
  "scopes": [
    "agent-factory:agents:agent-abc-123",
    "llm:models:gpt-4o",
    "llm:models:gpt-4o-mini",
    "storage:vector_stores:kb_xyz",
    "orgs:acme"
  ]
}
```

Wildcards are supported: `agent-factory:agents:*` grants access to all agents, `*` grants access to all resources.

If scopes are empty or omitted, the key has access to all resources within its permissions.

The API key editor in the AI Governance UI uses the same hierarchical PermissionTree component as the role editor, with scopes managed inline per resource.

## Cross-Workspace Validation

When other workspaces receive a request with an `iak_*` token, they validate it against AI Governance:

```
POST v1/auth/apikey
```

This internal endpoint (called via workspace JWT) returns the full API key context:

```json
{
  "error": false,
  "keyId": "key_xyz",
  "keyName": "Production Integration",
  "keyPrefix": "iak_acme-corp_a1b2c3d4",
  "org": {
    "slug": "acme-corp"
  },
  "permissions": ["llm:completions:use", "agents:read"],
  "scopes": ["models:gpt-4o"],
  "subscriptionLimits": [...],
  "llm": { ... }
}
```

The validation result is typically cached by calling workspaces to minimize cross-workspace calls.

## Usage Tracking

API key usage is tracked through the `v1/usage` endpoints and recorded via `on-apikey-used` events. Each key has an associated usage counter that can be checked against quota limits defined in the organization's subscription.

## Security

- Keys are stored **hashed** — plaintext values are never persisted
- Rotation provides **zero-downtime key renewal** — the new key is active immediately
- Expired keys are automatically rejected during validation
