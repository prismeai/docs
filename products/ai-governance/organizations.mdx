Organizations are the top-level tenant boundary in the platform. Every user belongs to one or more organizations, and all resources (agents, knowledge bases, API keys) are scoped to an organization.

## Organization Endpoints

### List Organizations

```
GET v1/orgs
```

Returns all organizations the authenticated user has access to. Platform admins see all organizations.

### Get Active Organization

```
GET v1/orgs/active
```

Returns the user's currently active organization (based on `user.meta.currentOrg`).

### Create Organization

```
POST v1/orgs
```

```json
{
  "name": "Acme Corp",
  "slug": "acme-corp",
  "description": "Enterprise AI deployment"
}
```

The creating user is automatically assigned the `owner` role.

### Get / Update Organization

```
GET v1/orgs/:org_slug
PUT v1/orgs/:org_slug
```

Update supports changing name, description, and branding settings (logo, colors, fonts, custom CSS).

## Members

### List Members

```
GET v1/orgs/:org_slug/members
```

Returns all members with their roles and group memberships.

### Add Member

```
POST v1/orgs/:org_slug/members
```

```json
{
  "user_id": "user_abc123",
  "role": "editor"
}
```

### Update Member Role

```
PUT v1/orgs/:org_slug/members/:user_id
```

```json
{
  "role": "admin"
}
```

### Remove Member

```
DELETE v1/orgs/:org_slug/members/:user_id
```

### Invite Member

```
POST v1/orgs/:org_slug/members/invite
```

```json
{
  "email": "user@example.com",
  "role": "viewer"
}
```

Generates an invitation code that the user can redeem to join the organization.

## Invitations

### List Invitations

```
GET v1/orgs/:org_slug/invites
```

### Create Invitation

```
POST v1/orgs/:org_slug/invites
```

```json
{
  "code": "welcome-2024",
  "role": "editor",
  "max_uses": 50,
  "expires_at": "2025-12-31T23:59:59Z"
}
```

### Delete Invitation

```
DELETE v1/orgs/:org_slug/invites/:invite_code
```

### Join Organization

```
POST v1/join
```

```json
{
  "invite_code": "welcome-2024"
}
```

Adds the authenticated user to the organization with the role specified in the invitation.

## Roles

Roles define what permissions a user has within an organization. There are built-in roles and custom roles.

### Built-in Roles

| Role | Description |
|------|-------------|
| `owner` | Full organization management |
| `admin` | Organization administration (cannot delete org) |
| `editor` | Create and manage resources |
| `viewer` | Read-only access |

### Custom Roles

```
GET    v1/orgs/:org_slug/roles
POST   v1/orgs/:org_slug/roles
GET    v1/orgs/:org_slug/roles/:role_slug
PUT    v1/orgs/:org_slug/roles/:role_slug
DELETE v1/orgs/:org_slug/roles/:role_slug
```

Create roles with specific permissions:

```json
{
  "slug": "analyst",
  "name": "Data Analyst",
  "permissions": [
    "agents:read",
    "agents:use",
    "insights:read",
    "collections:read",
    "collections:write"
  ]
}
```

## Groups

Groups allow organizing users for bulk access control. Agents can be bound to groups (via [Agent Factory access control](/products/agent-factory/agents-api#access-control)), giving all group members access.

### CRUD

```
GET    v1/orgs/:org_slug/groups
POST   v1/orgs/:org_slug/groups
GET    v1/orgs/:org_slug/groups/:group_slug
PUT    v1/orgs/:org_slug/groups/:group_slug
DELETE v1/orgs/:org_slug/groups/:group_slug
```

### Group Members

```
GET    v1/orgs/:org_slug/groups/:group_slug/members
POST   v1/orgs/:org_slug/groups/:group_slug/members
DELETE v1/orgs/:org_slug/groups/:group_slug/members/:user_id
```

When a group is deleted, the `iam.group.deleted` event is emitted, and Agent Factory automatically cleans up all agent bindings referencing that group.

## Join Rules

Join rules allow automatic organization membership based on email domain or other criteria:

```json
{
  "type": "email_domain",
  "value": "acme.com",
  "role": "viewer",
  "auto_approve": true
}
```

When a user signs up with a matching email domain, they are automatically added to the organization with the specified role.

Changes to join rules trigger the `joinrules.sync` event for cache invalidation.

## Users

```
GET v1/users
GET v1/users/:user_id
PUT v1/users/:user_id
```

User endpoints allow platform admins to list, view, and update user profiles across all organizations.

## IAM Membership

```
POST v1/iam/memberships
```

Returns all organization memberships for the authenticated user.

```
POST v1/iam/switch-org
```

```json
{
  "orgSlug": "acme-corp"
}
```

Switches the user's active organization. Updates `user.meta.currentOrg`.
