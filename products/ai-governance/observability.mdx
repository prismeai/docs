AI Governance provides a comprehensive observability layer for monitoring the entire platform ‚Äî workspace health, LLM costs, usage patterns, and error tracking.

## Platform Summary

```
GET v1/observability/platform/summary
```

Returns aggregated metrics across all workspaces:
- Total workspaces, active workspaces
- Total LLM requests, tokens consumed
- Total cost across all providers
- Error rate and top error types
- Active users and sessions

## Workspace Metrics

All workspace-level endpoints follow the pattern:

```
GET v1/observability/:workspace_id/{metric}
```

### General Metrics

```
GET v1/observability/:workspace_id/metrics
```

| Param | Type | Description |
|-------|------|-------------|
| `period` | string | Time period (`1h`, `24h`, `7d`, `30d`) ‚Äî default `24h` |

Data is always at hourly granularity (from pre-aggregated `platform_metrics`). Returns time-series data:
- Request count and latency
- LLM token usage (input/output)
- Active users
- Automation execution count

### Cost Tracking

```
GET v1/observability/:workspace_id/costs
```

Returns cost breakdown:
- Total cost (USD)
- Cost per LLM provider
- Cost per model
- Cost trend over time
- Carbon footprint estimates

### Usage Analytics

```
GET v1/observability/:workspace_id/usage
```

Returns usage patterns:
- Requests per hour/day
- Peak usage times
- User activity distribution
- Most-used automations

### Error Monitoring

```
GET v1/observability/:workspace_id/errors
```

Returns error analysis:
- Error count by type
- Error rate over time
- Most frequent error messages
- Stack traces (when available)

### Health Status

```
GET v1/observability/:workspace_id/health
```

Returns workspace health indicators:
- Health score (0-100) and status (`healthy` ‚â•90, `degraded` ‚â•70, `warning` ‚â•50, `critical` <50)
- Score breakdown with three components:
  - **Success score** (40%): `successRate √ó 40`
  - **Error score** (30%): `(1 - cappedErrorRate √ó 2) √ó 30` ‚Äî error rate capped at 0.5
  - **Latency score** (30%): `(1 - p95_ms / 5000) √ó 30` ‚Äî floored at 0
- Latency percentiles (p50, p95, p99)
- Error rate with severity thresholds
- Auto-detected issues (high error rate, high latency, low activity)

<Note>
The platform-level aggregation job (`jobs/aggregate-platform-metrics`) uses a slightly different formula: `(successRate √ó 40) + ((1 - errorRate √ó 10) √ó 30) + (latencyScore √ó 30)`, with thresholds `<40` critical, `<70` degraded, else healthy (no "warning" tier).
</Note>

### Activity Feed

```
GET v1/observability/:workspace_id/feed
```

Returns a chronological feed of recent workspace events ‚Äî deployments, configuration changes, errors, and notable metrics changes.

### Issues Detection

```
GET v1/observability/:workspace_id/issues
```

Automatically detected issues:
- Latency spikes
- Error rate increases
- Unusual usage patterns
- Budget threshold warnings

### Comparison

```
GET v1/observability/:workspace_id/compare
```

Compare metrics between time periods or between workspaces.

### Dependency Map

```
GET v1/observability/:workspace_id/dependencies
```

Returns the workspace's dependency graph ‚Äî which other workspaces it calls and which call it. Includes health status for each dependency.

### Distributed Tracing

```
GET v1/observability/:workspace_id/traces/:correlation_id
```

Trace a single request across workspace boundaries. Returns the full execution path with timing for each step.

## Data Aggregation

Metrics are aggregated by scheduled jobs:

| Schedule | Job | Purpose |
|----------|-----|---------|
| `15 * * * *` | `jobs/aggregate-platform-metrics` | Hourly aggregation at 3 levels: platform, per-org, per-workspace |
| On-demand | `v1/admin/refresh-metrics` | Force-refresh metrics (admin only) |

Aggregated data is stored in the `platform_metrics` collection.

## Internal Endpoints

For cross-workspace use:

```
POST v1/internal/search-events
```

Allows other workspaces to search events (logs) through Governance rather than directly querying the Events service. This centralizes access control for audit data.
