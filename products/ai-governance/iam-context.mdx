The `/me` endpoint is the bridge between AI Governance and the platform frontend. It provides everything the shell needs to render: the active organization context, navigation, branding, permissions, and user data.

## Active Organization

Every authenticated user has an **active organization** that scopes which products, permissions, and resources they can access via their organization role. The active org is resolved on each request with the following priority:

1. **URL path** — If the request targets `/v2/orgs/:orgSlug/...`, that org is used
2. **Session** — The org last selected by the user (persisted in the server-side session)
3. **First membership** — Falls back to the user's first organization

If the session references an org the user is no longer a member of, it is automatically reset to the first available org.

### Switching Organizations

For users with multiple organizations, the frontend calls:

```
PUT /v2/user/active-org
```

```json
{
  "orgSlug": "beta-inc"
}
```

This updates the server-side session, invalidates the user cache, and returns the new org's full context (navigation, branding, permissions, role). The frontend then reloads. See the [API Reference](/api-reference) for details.

The organizations list from the `/me` response populates the org switcher dropdown in the sidebar.

## The `/me` Endpoint

```
GET /v2/me
```

This is the first API call the platform shell makes after authentication. It returns the user profile merged with the active organization's context:

```json
{
  "id": "user_abc",
  "email": "user@acme.com",
  "org": {
    "slug": "acme-corp",
    "name": "Acme Corp",
    "status": "active",
    "branding": {
      "logo": { "light": "https://...", "dark": "https://..." },
      "colors": { "primary": "#1b4abb", "secondary": "#6c757d" },
      "fonts": { "heading": "Inter", "body": "Inter" },
      "customCssUrl": "https://..."
    },
    "navigation": {
      "version": "2",
      "menu": [
        {
          "id": "products",
          "type": "group",
          "label": "AI Products",
          "items": [
            { "id": "secure-chat", "label": "SecureChat", "icon": "MessageSquare", "href": "/" },
            { "id": "agent-factory", "label": "Agent Factory", "icon": "Bot", "href": "/apps/agent-factory", "permissions": ["agent-builder:*"] }
          ]
        }
      ]
    },
    "role": {
      "slug": "admin",
      "name": "Administrator",
      "permissions": ["secure-chat:*", "agent-factory:*", "orgs:members:manage", "storage:*"],
      "scopes": ["*"]
    },
    "groups": ["group_engineering_abc"],
    "settings": { "llm": { ... } },
    "subscription": { ... }
  },
  "permissions": ["secure-chat:*", "agent-factory:*", "orgs:members:manage", "storage:*"],
  "organizations": [
    { "slug": "acme-corp", "name": "Acme Corp", "role": "admin" },
    { "slug": "beta-inc", "name": "Beta Inc", "role": "user" }
  ],
  "sessionId": "sess_xyz",
  "expires": "2026-04-18T12:00:00Z"
}
```

The `org` object contains everything scoped to the active organization: branding, navigation, the user's role (with permissions and scopes), group memberships, settings, and subscription.

## Navigation Configuration

The sidebar navigation is fully dynamic — defined per organization and served within `org.navigation` in the `/me` response. The navigation is **server-side filtered** based on the user's role permissions before being returned.

For full details on menu item structure, permission gating, features, and the visual editor, see the [Menu Builder](/products/ai-governance/menu-builder) documentation.

## Branding

Organizations can customize the platform appearance:

| Setting | Description |
|---------|-------------|
| `version` | Branding schema version |
| `platformName` | Custom platform name (replaces "Prisme.ai") |
| `description` | Platform description |
| `logo.light` | Logo URL for light theme |
| `logo.dark` | Logo URL for dark theme |
| `favicon` | Favicon URL |
| `colors.primary` | Primary brand color |
| `colors.secondary` | Secondary brand color |
| `colors.accent` | Accent color |
| `fonts.heading` | Heading font family |
| `fonts.body` | Body text font family |
| `customCssUrl` | URL to a custom CSS file for advanced theming |
| `customCssInline` | Inline CSS string |

Branding is applied by the `IAMProvider` in the platform shell. CSS variables are set on the root element, and the custom CSS file (if any) is injected as a `<link>` tag.

## Onboarding

When a user has no organization membership, the platform shows an **onboarding modal**:

1. **Join with invite code** — Enter a code shared by an organization admin
2. **Create organization** — Available only to users with `PlatformAdmin` role
3. **Auto-join** — If [join rules](/products/ai-governance/joining-organizations#join-rules) match the user's attributes, they're automatically added

The onboarding flow is handled entirely by the frontend using the `/me` response.

## Degraded Mode

If no AI Governance workspace is configured (or it's unreachable), the platform shell falls back to **degraded mode**:

- Only the Builder link is shown in the sidebar
- No branding customization
- No organization context
- Basic user info from the OIDC session

This ensures the platform remains usable for development even without the governance layer.
**You still can access native governance features from /apps/ai-governance url.**
