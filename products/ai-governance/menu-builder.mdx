The platform sidebar navigation is fully dynamic and configured per organization. It is stored on the Organization object, served via the [IAM Context](/products/ai-governance/iam-context) endpoint, and rendered by the platform shell.

## Menu Item Structure

Each navigation item supports the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier |
| `type` | string | `category` (expandable group), `item` (link), or `divider` (visual separator) |
| `label` | string | Display label |
| `icon` | string | Lucide icon key (e.g. `bot`, `shield`, `folder`) |
| `color` | string | Hex color for icon/label (e.g. `#ef4444`) |
| `href` | string | URL path for items |
| `target` | string | `_self` (same tab) or `_blank` (new tab) |
| `expanded` | boolean | Whether a category starts expanded |
| `permissions` | string[] | Required permissions — item hidden if user lacks all of them |
| `badge` | string | Short label shown as a chip (e.g. "New", "Beta") |
| `features` | array | Agent or template features attached to the item |
| `visibility.tiers` | string[] | Subscription tiers that can see this item |
| `visibility.features` | string[] | Feature flags required |
| `items` | array | Nested children (for categories only) |

## Item Types

### Category

A collapsible group that contains child items. Categories can be expanded or collapsed by default. If all child items are filtered out by permissions, the category itself is hidden.

### Item

A navigable link with an `href`. Items can have an icon, badge, and attached features.

### Divider

A visual separator between items. Dividers have no interactive behavior — leading, trailing, and consecutive dividers are automatically cleaned up during rendering.

## Permission Gating

Navigation items are filtered based on the user's role permissions at two levels:

**Server-side:** The [IAM Context](/products/ai-governance/iam-context) endpoint filters the navigation before returning it, removing items the user cannot access.

**Client-side:** The platform shell applies the same filtering logic symmetrically:

- Items **with** `permissions` are shown if the user matches **any** listed permission (OR logic). Wildcard matching applies — `agent-factory:*` matches `agent-factory:agents:read`.
- Items **without** `permissions` (empty or omitted) are only shown to users with the `*` wildcard permission (owners/admins).
- Categories are hidden if they have zero visible children after filtering.
- Features within items are also filtered by their own `permissions` array.

## Features

Menu items can have attached features that appear as sub-entries:

### Agent Features

```json
{
  "type": "agent",
  "agentId": "agent_abc123",
  "slug": "my-agent",
  "label": "My Agent",
  "permissions": ["agent-factory:agents:read"]
}
```

Agent features link directly to an agent conversation within the parent menu item's context.

### Template Features

```json
{
  "type": "template",
  "templateId": "tpl_xyz",
  "slug": "weekly-report",
  "label": "Weekly Report",
  "permissions": ["agent-factory:agents:read"]
}
```

Template features provide pre-configured entry points for specific agent workflows.

## Menu Editor UI

The AI Governance UI includes a visual menu editor (requires `orgs:navigation:manage` permission):

- **Drag and drop** — Reorder items by dragging. Items are inserted after the drop target.
- **Nested hierarchy** — Items can be assigned to a parent category via the edit modal's level selector.
- **Icon picker** — Choose from ~150 Lucide icons organized by category (navigation, business, tech, communication, security, charts, etc.) with 9 color options (default, red, orange, amber, green, teal, blue, purple, pink).
- **Edit modal** — Configure label, icon + color, href, target, level (parent category), badge, permissions, and features.
- **JSON import/export** — Export the full navigation as a `.json` file. Import replaces the current navigation from a JSON file (max 1MB).
- **Unsaved changes** — An indicator shows when changes haven't been saved, with a browser warning on navigation.

## Default Navigation

When no navigation is configured on the organization (or AI Governance is unreachable), the platform falls back to **degraded mode** — only a single "Builder" link is shown in the sidebar. See [IAM Context > Degraded Mode](/products/ai-governance/iam-context#degraded-mode) for details.
