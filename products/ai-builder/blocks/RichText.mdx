---
title: "RichText Block"
description: "Display formatted text with support for Markdown and HTML."
---

## Minimal usage

```yaml
# Page/Block
- slug: RichText
  content: Hello World
```

`content` can be raw text, localized string, markdown or html. But only a few html tags will be allowed without `allowUnsecure` flag.

```yaml
# Page/Block
- slug: RichText
  content:
    en: Hello World
    fr: Bonjour monde
```

```yaml
# Page/Block
- slug: RichText
  content: |-
    Hello **world**
```

```yaml
# Page/Block
- slug: RichText
  content: <div><p>Hello <strong>World</strong></p></div>
```

## Advanced usage

`allowUnsecure` let all html tag to be displayed.

```yaml
# Page/Block
- slug: RichText
  content: |-
    <div class="my-class">
      <p>Hello <strong>World</strong>
      <img src="my-img.png" />
    </div>
  allowUnsecure: true
```

`markdown` can be disabled by setting this attribute to false.

```yaml
# Page/Block
- slug: RichText
  content: |-
    Hello **world**
  markdown: false
```

will not display "world" in a strong tag.

`tag` is an option to set the nature of the container tag. By default, it's a div tag.

```yaml
# Page/Block
- slug: RichText
  content: Hello World
  tag: h2
```
