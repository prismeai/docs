## Prerequisites

- AWS account with permissions to create EKS, RDS, S3, ElastiCache, and OpenSearch resources
- AWS CLI installed and configured
- Familiarity with Kubernetes and AWS services

## AWS Resources

<AccordionGroup>
  <Accordion title="Kubernetes (EKS)">
    - 3-5 nodes, 4 vCPU / 16GB RAM each
    - Autoscaling enabled, multi-AZ

    ```bash
    eksctl create cluster --name=prisme-ai-cluster \
      --version=1.28 \
      --nodegroup-name=standard-workers \
      --nodes=3 --nodes-min=3 --nodes-max=5 \
      --node-type=t3.xlarge \
      --region=eu-west-1
    ```
  </Accordion>
  <Accordion title="Database (RDS)">
    - Amazon RDS for PostgreSQL, DocumentDB, or self-managed MongoDB
    - 3-node cluster, `db.m5.large` minimum
    - Multi-AZ, automated backups
  </Accordion>
  <Accordion title="Elasticsearch / OpenSearch">
    - Amazon OpenSearch Service
    - 3 nodes, `m6g.large.search`
    - Encryption at rest and node-to-node
  </Accordion>
  <Accordion title="Redis (ElastiCache)">
    - Amazon ElastiCache for Redis
    - 3-node cluster, `cache.m6g.large`
    - Multi-AZ failover
  </Accordion>
  <Accordion title="Object Storage (S3)">
    - Buckets for models, uploads, and public assets
    - CloudFront CDN for public assets
    - Bucket versioning and lifecycle policies
  </Accordion>
  <Accordion title="File System (EFS)">
    - Amazon EFS for shared Kubernetes storage
    - General Purpose performance mode
    - Security groups limiting access to K8s nodes
  </Accordion>
</AccordionGroup>

## Deployment

<Steps>
  <Step title="Create EKS Cluster">
    ```bash
    eksctl create cluster --name=prisme-ai --region=eu-west-1
    ```
  </Step>
  <Step title="Provision Databases">
    Set up RDS/DocumentDB, OpenSearch, ElastiCache, S3, and EFS.
  </Step>
  <Step title="Configure DNS">
    Route 53 entries:
    - `api.yourdomain.com` — API Gateway
    - `platform.yourdomain.com` — Platform frontend
    - `*.pages.yourdomain.com` — Workspace pages
  </Step>
  <Step title="Deploy via Helm">
    ```bash
    helm repo add prismeai https://helm.prisme.ai/charts
    helm install prisme-core prismeai/prismeai-core --namespace prisme -f values.yaml
    ```
  </Step>
  <Step title="Deploy Platform Frontend">
    See [Platform Service](/self-hosting/platform-service) for building and deploying the new frontend.
  </Step>
  <Step title="Deploy Backend Workspaces">
    See [Workspace Deployment](/self-hosting/workspace-deployment) for pushing the 18 backend workspaces.
  </Step>
</Steps>

## Security Best Practices

<CardGroup cols="2">
  <Card title="IAM Roles" icon="shield-halved">
    Use IRSA (IAM Roles for Service Accounts) for K8s-to-AWS permissions. Audit roles regularly.
  </Card>
  <Card title="Network Isolation" icon="network-wired">
    Private subnets, VPC security groups, NAT gateways for controlled internet access.
  </Card>
  <Card title="Secrets Management" icon="key">
    AWS Secrets Manager or HashiCorp Vault. Rotate passwords and API keys regularly.
  </Card>
  <Card title="Monitoring" icon="eye">
    CloudTrail, CloudWatch, Prometheus/Grafana for comprehensive monitoring.
  </Card>
</CardGroup>
