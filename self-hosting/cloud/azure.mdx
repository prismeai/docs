## Azure Resources

<AccordionGroup>
  <Accordion title="Kubernetes (AKS)">
    - Azure Kubernetes Service
    - 3-5 nodes, Standard_D4s_v3 or equivalent
    - Availability zones enabled
  </Accordion>
  <Accordion title="Database">
    - Azure CosmosDB (MongoDB API) or Azure Database for PostgreSQL
    - 3-node configuration with zone redundancy
  </Accordion>
  <Accordion title="Redis (Azure Cache)">
    - Azure Cache for Redis, Premium tier
    - Zone redundancy enabled
    - Supports Microsoft Entra ID passwordless authentication via managed identity
  </Accordion>
  <Accordion title="Storage">
    - Azure Blob Storage for objects (models, uploads)
    - Azure Files for RWX shared filesystem
  </Accordion>
  <Accordion title="Elasticsearch">
    - Elastic Cloud on Azure or self-managed OpenSearch on AKS
  </Accordion>
</AccordionGroup>

## Microsoft Entra ID Integration

Azure deployments can use **passwordless authentication** with managed identities for Redis and PostgreSQL:

1. Create a user-assigned managed identity
2. Configure federated credentials for AKS
3. Grant the identity access to your databases

Key Helm values for passwordless auth:

```yaml
global:
  azureSystemIdentity: "your-managed-identity-client-id"
  azureManagedIdentityClientId: "your-managed-identity-client-id"
  permissions:
    user: "managed-identity-username"
```

See [Enterprise Authentication](/self-hosting/enterprise/authentication) for SSO configuration.

## Deployment

<Steps>
  <Step title="Create AKS Cluster">
    ```bash
    az aks create --resource-group prisme-rg --name prisme-aks \
      --node-count 3 --node-vm-size Standard_D4s_v3 \
      --zones 1 2 3
    ```
  </Step>
  <Step title="Provision Services">
    Set up CosmosDB/PostgreSQL, Azure Cache, Blob Storage, Azure Files.
  </Step>
  <Step title="Deploy via Helm">
    ```bash
    helm repo add prismeai https://helm.prisme.ai/charts
    helm install prisme-core prismeai/prismeai-core --namespace prisme -f values.yaml
    ```
  </Step>
  <Step title="Deploy Platform & Workspaces">
    See [Platform Service](/self-hosting/platform-service) and [Workspace Deployment](/self-hosting/workspace-deployment).
  </Step>
</Steps>
