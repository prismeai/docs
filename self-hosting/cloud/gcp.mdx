## GCP Resources

<AccordionGroup>
  <Accordion title="Kubernetes (GKE)">
    - Google Kubernetes Engine
    - 3-5 nodes, e2-standard-4 or equivalent
    - Regional cluster with multiple zones
  </Accordion>
  <Accordion title="Database">
    - Cloud SQL for PostgreSQL or self-managed MongoDB on GKE
    - High availability configuration
  </Accordion>
  <Accordion title="Redis (Memorystore)">
    - Cloud Memorystore for Redis
    - Standard tier with replica for HA
  </Accordion>
  <Accordion title="Storage">
    - Cloud Storage for objects (models, uploads)
    - Filestore for RWX shared filesystem
  </Accordion>
  <Accordion title="Elasticsearch">
    - Elastic Cloud on GCP or self-managed on GKE
  </Accordion>
</AccordionGroup>

## Deployment

<Steps>
  <Step title="Create GKE Cluster">
    ```bash
    gcloud container clusters create prisme-gke \
      --region=europe-west1 \
      --num-nodes=3 \
      --machine-type=e2-standard-4
    ```
  </Step>
  <Step title="Provision Services">
    Cloud SQL, Memorystore, Cloud Storage, Filestore.
  </Step>
  <Step title="Deploy via Helm">
    ```bash
    helm repo add prismeai https://helm.prisme.ai/charts
    helm install prisme-core prismeai/prismeai-core --namespace prisme -f values.yaml
    ```
  </Step>
  <Step title="Deploy Platform & Workspaces">
    See [Platform Service](/self-hosting/platform-service) and [Workspace Deployment](/self-hosting/workspace-deployment).
  </Step>
</Steps>

## Security

- Use Workload Identity for K8s-to-GCP IAM binding
- Cloud NAT for egress from private nodes
- Secret Manager for credentials
- Cloud Armor for DDoS protection
