## Prerequisites

- OpenShift 4.12+
- `oc` CLI installed and configured
- OperatorHub access for Elasticsearch/Redis operators

## Deployment

<Steps>
  <Step title="Create Project">
    ```bash
    oc new-project prisme
    ```
  </Step>
  <Step title="Deploy via Helm">
    ```bash
    helm repo add prismeai https://helm.prisme.ai/charts
    helm install prisme-core prismeai/prismeai-core --namespace prisme -f values.yaml
    ```
  </Step>
  <Step title="Configure Routes">
    ```bash
    oc create route edge api --service=prismeai-api-gateway --port=3001 \
      --hostname=api.yourdomain.com
    oc create route edge platform --service=prismeai-platform --port=80 \
      --hostname=platform.yourdomain.com
    ```
  </Step>
  <Step title="Deploy Platform & Workspaces">
    See [Platform Service](/self-hosting/platform-service) and [Workspace Deployment](/self-hosting/workspace-deployment).
  </Step>
</Steps>

## Security Considerations

- Use OpenShift's built-in SCC (Security Context Constraints) for pod security
- Leverage OpenShift's integrated OAuth for SSO
- Use NetworkPolicy for service isolation
- Ensure all routes use TLS edge termination
