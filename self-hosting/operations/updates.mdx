## Pre-Update Checklist

<Steps>
  <Step title="Backup">
    Create backups of all databases and configuration. See [Backup & Restore](/self-hosting/operations/backup).
  </Step>
  <Step title="Review Release Notes">
    Check for breaking changes, new dependencies, or configuration changes.
  </Step>
  <Step title="Plan Rollback">
    Document the current Helm revision and keep database backups accessible.
  </Step>
</Steps>

## Update Process

### Core Services (Helm)

```bash
# Update Helm repo
helm repo update

# Preview changes
helm diff upgrade prisme-core prismeai/prismeai-core -n prisme -f values.yaml

# Apply update
helm upgrade prisme-core prismeai/prismeai-core -n prisme -f values.yaml

# Verify pods are running
kubectl get pods -n prisme -w
```

### Platform Frontend

```bash
# Build new image
docker build -f services/platform/Dockerfile -t prisme-platform:new-version .

# Update deployment
kubectl set image deployment/platform platform=prisme-platform:new-version -n prisme
```

### Backend Workspaces

Use the Prisme.ai Builder MCP or CLI to push updated workspace configurations. See [Workspace Deployment](/self-hosting/workspace-deployment).

## Post-Update Verification

<Steps>
  <Step title="Health Check">
    ```bash
    curl https://api.yourdomain.com/v2/readiness \
      -H "Authorization: Bearer $ADMIN_TOKEN"
    ```
  </Step>
  <Step title="Run Tests">
    ```bash
    # Test each critical workspace
    curl -X POST https://api.yourdomain.com/v2/workspaces/slug:llm-gateway/webhooks/tests/run-all \
      -H "Authorization: Bearer $TOKEN"
    ```
  </Step>
  <Step title="Verify Functionality">
    - User login/logout
    - Agent creation and chat
    - File uploads
    - Workspace management
  </Step>
</Steps>

## Rollback

```bash
# View revision history
helm history prisme-core -n prisme

# Rollback to previous revision
helm rollback prisme-core <REVISION> -n prisme

# Verify
kubectl get pods -n prisme -w
```

For database rollback, restore from the pre-update backup.
