## Readiness Endpoint

The API Gateway provides a readiness endpoint that checks all services:

```bash
curl https://api.yourdomain.com/v2/readiness \
  -H "Authorization: Bearer $ADMIN_TOKEN"
```

<Info>
  Restricted to super admins and internal API keys. Results are cached for 30 seconds (configurable via `READINESS_CACHE_TTL_MS`).
</Info>

Response:

```json
{
  "status": "ready",
  "services": {
    "api-gateway": { "status": "ready", "modulesReady": ["broker", "permissions", "users"] },
    "runtime": { "status": "ready", "modulesReady": ["broker", "permissions", "collections"] },
    "events": { "status": "ready", "modulesReady": ["broker", "elasticsearch"] },
    "workspaces": { "status": "ready", "modulesReady": ["broker", "permissions", "storage"] }
  }
}
```

## Workspace Integration Tests

Each backend workspace includes test suites. Run them after deployment:

```bash
# LLM Gateway tests
curl -X POST https://api.yourdomain.com/v2/workspaces/slug:llm-gateway/webhooks/tests/run-all \
  -H "Authorization: Bearer $TOKEN"

# Storage tests
curl -X POST https://api.yourdomain.com/v2/workspaces/slug:storage/webhooks/tests/run-all \
  -H "Authorization: Bearer $TOKEN"

# Agent Factory tests
curl -X POST https://api.yourdomain.com/v2/workspaces/slug:agent-factory/webhooks/tests/run-all \
  -H "Authorization: Bearer $TOKEN"
```

See [Development — Testing](/development/testing) for the complete list of test suites by workspace.

## Manual Verification Checklist

After a new deployment or update, verify:

- [ ] User login and logout
- [ ] Anonymous login
- [ ] Workspace creation
- [ ] Agent creation and conversation
- [ ] File upload and retrieval
- [ ] Event streaming (WebSocket)
- [ ] Page rendering
- [ ] SSO login (if configured)
- [ ] API key creation and usage
- [ ] Workspace publishing and versioning

## Platform Frontend Health

The platform frontend (nginx) responds to any valid path:

```bash
curl https://platform.yourdomain.com/env-config.js
```

For Kubernetes probes:

```yaml
livenessProbe:
  httpGet:
    path: /env-config.js
    port: 80
  initialDelaySeconds: 5
  periodSeconds: 10
```
