Prisme.ai is composed of several microservices and infrastructure components. This page covers the infrastructure-level architecture. For the platform's workspace-based product architecture, see [Architecture Overview](/architecture/overview).

## Core Services

- **API Gateway** — Central access point for authentication, routing, and load balancing
- **Workspace Service** — Manages workspaces, permissions, DSUL storage, and metadata
- **Event Service** — Event-driven interactions, WebSocket streaming, and analytics storage
- **Runtime Service** — Executes automations and processes events
- **Platform Frontend** — New React SPA serving the platform UI (see [Platform Service](/self-hosting/platform-service))
- **Pages Service** — Renders end-user workspace pages

## Extended Services

- **Functions Service** — User-defined JavaScript or Python script execution
- **Crawler Service** — Indexes external web document content
- **Search Service** — Internal search (Redis Stack, Elasticsearch/OpenSearch)

## Databases

| Database | Purpose |
|----------|---------|
| MongoDB/PostgreSQL | Structured data: users, roles, permissions, workspace configs |
| Redis | Caching, real-time streaming, message brokering, OIDC sessions |
| Elasticsearch/OpenSearch | Event storage, document indexing, analytics |
| Vector Store (OpenSearch/Redis Stack) | Vectorized data for RAG and AI Knowledge |

## File and Object Storage

- **Filesystem** — RWX Kubernetes PVC for local file storage
- **Object Storage** — S3, Azure Blob, or GCS for uploads, models, and documents

## Deployment Patterns

### Kubernetes (Recommended)

Scalable and resilient production deployments using Helm charts:

```bash
helm repo add prismeai https://helm.prisme.ai/charts
helm install prisme-core prismeai/prismeai-core --namespace prisme -f values.yaml
```

### Docker Compose

For development or small-scale environments:

```bash
docker-compose up -d
```

See [Docker deployment](/self-hosting/kubernetes/docker) for details.

## Network Architecture

- **Ingress Controller** — Required for routing traffic to services
- **TLS/SSL** — Required for all external and recommended for internal communication
- **Network Policies** — Recommended to secure inter-service communication
- Only the API Gateway is publicly exposed; all other services run in a private network

## High Availability

- **Replica Sets** — Multiple instances of each microservice for fault tolerance
- **HPA** — Horizontal Pod Autoscaling based on workload
- **Persistent Storage** — Distributed file systems or cloud storage

See [High Availability](/self-hosting/kubernetes/high-availability) for detailed configuration.
