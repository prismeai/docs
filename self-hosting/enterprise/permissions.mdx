Prisme.ai implements a multi-layer authorization model.

## Authorization Layers

### 1. Super Admins

Platform-wide administrators configured via environment variable or Helm:

```bash
# Environment variable
SUPER_ADMIN_EMAILS=admin@company.com,admin2@company.com
```

```yaml
# Helm values
prismeai-api-gateway:
  config:
    admins:
      - admin@company.com
```

Super admins can:
- Access all workspaces
- Manage users and organizations
- Configure platform settings
- Override rate limits

### 2. Workspace Roles

Each workspace defines its own RBAC rules in `security.yml`:

```yaml
authorizations:
  roles:
    editor: {}
    viewer:
      auth:
        prismeai: {}
    api:
      auth:
        apiKey: {}

rules:
  - role: editor
    action: manage
    subject: workspaces

  - role: viewer
    action: read
    subject: workspaces

  - role: api
    action:
      - read
      - create
    subject: files
```

### 3. SSO-Based Access

Assign roles based on SSO attributes:

```yaml
authorizations:
  roles:
    admin:
      auth:
        MySSOProvider:
          groups:
            - platform-admins
    user:
      auth:
        MySSOProvider:
          groups:
            - platform-users
```

### 4. Organization-Level Roles

AI Governance manages organization membership with built-in and custom roles. See [AI Governance — Organizations](/products/ai-governance/organizations) for the full RBAC model.

## Product-Level Permissions

Each product workspace defines specific permissions through its `security.yml`. Common patterns:

| Product | Key Roles | Access Control |
|---------|-----------|---------------|
| Agent Factory | org member, API key | Agents scoped to organizations |
| AI Governance | super admin, org admin | Full platform IAM |
| AI Collection | org member | Agents limited to their org's data |
| AI Insights | org admin | Analytics restricted to org scope |
| LLM Gateway | workspace JWT | Service-to-service only |
| Storage | workspace JWT | Service-to-service only |

## API Key Permissions

API keys carry workspace-scoped RBAC rules:

```json
{
  "rules": [
    { "action": ["read"], "subject": "workspaces" },
    { "action": ["create", "read"], "subject": "files" }
  ]
}
```

See [AI Governance — API Keys](/products/ai-governance/api-keys) for the platform API key system.
