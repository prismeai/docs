Enterprise features extend the core Prisme.ai platform with additional microservices and integrations.

## Extended Services

| Service | Description |
|---------|-------------|
| `prismeai-functions` | Execute user-defined JavaScript and Python scripts |
| `prismeai-crawler` | Index and manage external web documents |
| `prismeai-searchengine` | Internal search capabilities |
| `prismeai-llm` | Local LLM inference (LocalAI, Ollama) |

These services are deployed separately from the core stack, typically via the `prismeai/prismeai-apps` Helm chart in a dedicated namespace:

```bash
helm install prisme-apps prismeai/prismeai-apps --namespace prisme-apps -f apps-values.yaml
```

<Info>
  Extended service Docker images require GitLab credentials. Contact support for access.
</Info>

## Troubleshooting

<Accordion title="ImagePullBackOff">
  Verify your GitLab registry credentials are configured as a Kubernetes secret and referenced in the service account.
</Accordion>

<Accordion title="CrashLoopBackOff">
  Check pod logs: `kubectl logs <pod> -n prisme-apps --previous`. Common causes: missing config, unreachable dependencies.
</Accordion>

<Accordion title="Service Connectivity">
  Ensure the `INTERNAL_API_URL` environment variable is set correctly and network policies allow cross-namespace communication.
</Accordion>
