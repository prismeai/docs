The Functions microservice enables user-defined script execution within workspace automations.

## Supported Runtimes

<Tabs>
  <Tab title="Python">
    | Variable | Description | Default |
    |----------|-------------|---------|
    | `PYTHON_FUNCTIONS_RUN_TIMEOUT` | Execution timeout (ms) | 30000 |
    | `PYTHON_API_URL` | Python service URL | |
    | `KERNEL_POOL_SIZE` | Pre-warmed kernel pool | 3 |
    | `PYTHON_BUILTIN_MODULES` | Allowed Python modules | (see below) |

    **Allowed modules**: `json`, `re`, `datetime`, `math`, `random`, `hashlib`, `base64`, `urllib`, `html`, `collections`, `itertools`, `functools`, `operator`, `string`, `textwrap`, `unicodedata`, `io`, `csv`, `typing`, `dataclasses`, `enum`, `copy`, `pprint`, `statistics`, `decimal`, `fractions`

    **Excluded for security**: `os`, `sys`, `subprocess`, `shutil`, `importlib`, `ctypes`, `socket`
  </Tab>
  <Tab title="JavaScript (Node.js)">
    | Variable | Description | Default |
    |----------|-------------|---------|
    | `FUNCTIONS_RUN_TIMEOUT` | Execution timeout (ms) | 30000 |
    | `NODEJS_BUILTIN_MODULES` | Allowed Node.js modules | (restricted set) |
    | `NPM_CONFIG_REGISTRY` | Custom npm registry URL | |

    JavaScript functions run in a sandboxed VM environment.
  </Tab>
</Tabs>

## Usage in Automations

```yaml
do:
  - functions:
      script: |
        const result = data.items.map(item => ({
          name: item.name.toUpperCase(),
          score: item.value * 1.5
        }));
        return { processed: result };
      args:
        items: '{{input.items}}'
      output: processedData
```

## Advanced Features

- **Async execution**: Functions can return Promises
- **Shared context**: Access workspace variables via the `context` object
- **Shared cache**: In-memory `cache` object persists across executions within the same Runtime instance
- **Dependencies**: Install npm packages via `NPM_CONFIG_REGISTRY` for custom registries

## Testing

```bash
curl -X POST https://api.yourdomain.com/v2/workspaces/<ID>/webhooks/test-function \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"items": [{"name": "test", "value": 10}]}'
```

<Warning>
  Functions run in a restricted environment. File system access, network sockets, and process spawning are disabled for security.
</Warning>
