title: "Local LLMs"

The `prismeai-llm` service enables local LLM inference, keeping all data on-premises.

## Supported Backends

<Tabs>
  <Tab title="LocalAI (Default)">
    Supports GGUF model files with OpenAI-compatible API.

    <Steps>
      <Step title="Place Model Files">
        Copy your `.gguf` model files and configuration to the models directory.
      </Step>
      <Step title="Create Model Config">
        ```yaml
        # models/my-model.yaml
        name: my-model
        backend: llama
        parameters:
          model: /models/my-model.gguf
          temperature: 0.7
          top_p: 0.9
        ```
      </Step>
      <Step title="Configure Helm">
        ```yaml
        prismeai-llm:
          models:
            - name: my-model
              file: my-model.gguf
        ```
      </Step>
    </Steps>
  </Tab>
  <Tab title="Ollama">
    Alternative backend using Ollama's model management.

    ```bash
    # Pull a model
    ollama pull mistral

    # Verify
    ollama list
    ```

    Configure in Helm values:

    ```yaml
    prismeai-llm:
      image: ollama/ollama:latest
      command: ["ollama", "serve"]
    ```
  </Tab>
</Tabs>

## Integration with LLM Gateway

Once `prismeai-llm` is running, register it as a provider in the LLM Gateway workspace:

```yaml
providers:
  local:
    type: openai
    endpoint: http://prismeai-llm:8080/v1
    api_key: "not-needed"
```

Then define models in the LLM Gateway's model catalog that route to this provider.

## Performance Notes

| Model | Approx. Load Time | Hardware |
|-------|-------------------|----------|
| Phi-2 (2.7B) | ~3 min | 16 CPU |
| Mistral 7B | ~10 min | 16 CPU |

- GPU acceleration significantly improves inference speed
- NVIDIA GPUs: Set `runtimeClassName: nvidia` in the pod spec
- Models load into memory on first request

<Warning>
  Changing embedding models requires reindexing all documents — vector dimensions may differ between models.
</Warning>

## Testing

```bash
# Test chat completion
curl http://prismeai-llm:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "my-model",
    "messages": [{"role": "user", "content": "Hello"}]
  }'

# Test embeddings
curl http://prismeai-llm:8080/v1/embeddings \
  -H "Content-Type: application/json" \
  -d '{
    "model": "my-embedding-model",
    "input": "Test text for embedding"
  }'
```
