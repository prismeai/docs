## Infrastructure Components

### Kubernetes Cluster

5 nodes with 8GB RAM and 4 vCPU each.

### Storage

<Tabs>
  <Tab title="EFS/Shared Filesystem">
    50GB elastic file system for shared storage.
  </Tab>
  <Tab title="Object Storage">
    Per environment:
    - 1 "models" bucket
    - 1 "uploads" bucket
    - 1 "uploads-public" bucket (behind CDN)
  </Tab>
</Tabs>

### Databases

<Tabs>
  <Tab title="MongoDB/PostgreSQL">
    - **Data**: RBAC, users, application data
    - **Config**: 3-node replica set, 2GB RAM / 2 vCPU, 1,000 IOPS
    - **Disk**: 10GB
    - **Version**: MongoDB 6+ or PostgreSQL 10+
  </Tab>
  <Tab title="Redis">
    - **Data**: Event streams, permission cache, OIDC sessions, rate limits
    - **Config**: 1 master + 2 replicas, 3GB RAM / 2 vCPU per node
    - **Version**: Redis 5+
  </Tab>
  <Tab title="Elasticsearch/OpenSearch">
    - **Data**: Persisted events, document content
    - **Config**: 3 nodes, 8GB RAM / 4 vCPU
    - **Disk**: 400GB per node (NVMe/SSD)
    - **Version**: 8+
  </Tab>
  <Tab title="Vector Database">
    - **Implementation**: Redis Stack or OpenSearch
    - **Data**: Text chunks with vector embeddings
    - **Config**: 1 master + 2 replicas, 5GB RAM / 2 vCPU
    - **Scaling**: 100K chunks from 20K documents ≈ 2.6GB RAM
  </Tab>
</Tabs>

## Scaling Considerations

<Accordion title="Horizontal vs. Vertical">
  - **Horizontal**: Add nodes to distribute load (recommended for K8s and DB replicas)
  - **Vertical**: Increase resources on existing nodes (for temporary peaks)

  Combine both approaches for production.
</Accordion>

<Accordion title="Resource Allocation">
  1. Start with baseline recommendations
  2. Monitor utilization during testing
  3. Identify bottlenecks (usually memory or disk I/O)
  4. Scale constrained resources first

  Vector databases and Elasticsearch benefit most from additional memory. Redis is often CPU-constrained.
</Accordion>

<Accordion title="Environment Isolation">
  - **Development**: Shared infrastructure, minimal isolation
  - **Staging**: Mimic production with smaller resources
  - **Production**: Dedicated resources, full isolation
</Accordion>
