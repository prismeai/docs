Helm is the recommended way to deploy Prisme.ai on Kubernetes.

## Installation

<Steps>
  <Step title="Add Helm Repository">
    ```bash
    helm repo add prismeai https://helm.prisme.ai/charts
    helm repo update
    ```
  </Step>
  <Step title="Get Default Values">
    ```bash
    helm pull prismeai/prismeai-core --untar
    cd prismeai-core
    # Edit values.yaml with your configuration
    ```
  </Step>
  <Step title="Create Namespace">
    ```bash
    kubectl create namespace prisme
    ```
  </Step>
  <Step title="Install">
    ```bash
    helm install prisme-core prismeai/prismeai-core \
      --namespace prisme \
      -f values.yaml
    ```
  </Step>
</Steps>

## Key Configuration

### PostgreSQL

If using PostgreSQL instead of MongoDB, configure these values:

```yaml
global:
  storage:
    permissions:
      driver: postgresql
      host: "postgresql://user:pass@host:5432/permissions"

prismeai-api-gateway:
  storage:
    users:
      driver: postgresql
      host: "postgresql://user:pass@host:5432/users"
      user: "db_user"

prismeai-runtime:
  storage:
    collections:
      driver: postgresql
      host: "postgresql://user:pass@host:5432/collections"
      user: "db_user"
```

### External Services

Point to your managed services:

```yaml
global:
  redis:
    host: "redis://your-redis-host:6379"
  elasticsearch:
    host: "https://your-elasticsearch-host:9200"
```

## Common Commands

```bash
# Upgrade after config changes
helm upgrade prisme-core prismeai/prismeai-core -n prisme -f values.yaml

# Check status
helm status prisme-core -n prisme

# View release history
helm history prisme-core -n prisme

# Rollback to previous version
helm rollback prisme-core 1 -n prisme

# Uninstall
helm uninstall prisme-core -n prisme
```

## Troubleshooting

<Accordion title="Pods stuck in CrashLoopBackOff">
  Check logs for the failing pod:
  ```bash
  kubectl logs <pod-name> -n prisme --previous
  ```
  Common causes: incorrect database connection strings, missing secrets, insufficient memory.
</Accordion>

<Accordion title="Service connectivity issues">
  Verify services can reach each other:
  ```bash
  kubectl exec -it <api-gateway-pod> -n prisme -- curl http://prismeai-runtime:3003/
  ```
</Accordion>
