Docker Compose deployment is ideal for local development and testing environments.

## Quick Start

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://gitlab.com/prisme.ai/prisme.ai.git
    cd prisme.ai
    ```
  </Step>
  <Step title="Configure Environment">
    Create a `.env` file in the root directory:

    ```bash
    PAGES_HOST=.pages.localhost:3100
    CONSOLE_URL=http://localhost:3000
    API_URL=http://localhost:3001/v2
    ```
  </Step>
  <Step title="Start Services">
    ```bash
    docker-compose up -d
    ```
  </Step>
  <Step title="Verify">
    ```bash
    docker-compose ps
    docker-compose logs -f api-gateway
    ```
  </Step>
</Steps>

## Services

The Docker Compose configuration starts the following services:

| Service | Port | Description |
|---------|------|-------------|
| api-gateway | 3001 | Authentication, routing |
| workspaces | 3002 | Workspace management |
| runtime | 3003 | Automation execution |
| events | 3004 | Event streaming |
| pages | 3100 | Page rendering |
| mongodb | 27017 | Database |
| redis | 6379 | Cache and broker |
| elasticsearch | 9200 | Event storage |

## Adding the Platform Frontend

The Docker Compose setup provides the core infrastructure services. To add the new platform frontend:

```bash
docker build -f services/platform/Dockerfile -t prisme-platform .
docker run -d -p 3000:80 \
  -e API_URL=http://localhost:3001/v2 \
  -e OIDC_STUDIO_CLIENT_ID=local-client-id \
  prisme-platform
```

See [Platform Service](/self-hosting/platform-service) for full Docker configuration.

## Common Operations

```bash
# Stop all services
docker-compose down

# View logs for a specific service
docker-compose logs -f runtime

# Restart a service
docker-compose restart api-gateway

# Remove all data (destructive)
docker-compose down -v
```

<Warning>
  Docker Compose is not recommended for production use. For production deployments, use [Kubernetes with Helm](/self-hosting/kubernetes/helm).
</Warning>
