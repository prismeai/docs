## Replica Counts

| Service | Minimum Replicas | Notes |
|---------|-----------------|-------|
| API Gateway | 2+ | Stateless, scales horizontally |
| Runtime | 3+ | Handles automation execution load |
| Platform Frontend | 2+ | Stateless nginx, very lightweight |
| Pages | 2+ | Stateless page rendering |
| Events | 2+ | WebSocket connections are sticky |

## Multi-Zone Deployment

Spread pods across availability zones:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: prismeai-api-gateway
spec:
  replicas: 3
  template:
    spec:
      topologySpreadConstraints:
        - maxSkew: 1
          topologyKey: topology.kubernetes.io/zone
          whenUnsatisfiable: DoNotSchedule
          labelSelector:
            matchLabels:
              app: prismeai-api-gateway
```

## Pod Disruption Budgets

Ensure minimum availability during updates:

```yaml
apiVersion: policy/v1
kind: PodDisruptionBudget
metadata:
  name: prismeai-api-gateway-pdb
spec:
  minAvailable: 1
  selector:
    matchLabels:
      app: prismeai-api-gateway
```

## Database HA

| Component | Recommendation |
|-----------|---------------|
| MongoDB | 3-node replica set with arbiter |
| PostgreSQL | Primary + standby with automated failover |
| Elasticsearch | 3 data nodes + 3 dedicated master nodes |
| Redis | Sentinel (3 nodes) or Redis Cluster |

## Platform Frontend HA

The platform frontend is stateless — simply increase replicas:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: platform
spec:
  replicas: 3
  template:
    spec:
      containers:
        - name: platform
          image: prisme-platform:latest
          resources:
            requests:
              memory: "64Mi"
              cpu: "50m"
            limits:
              memory: "128Mi"
              cpu: "200m"
```

See [Platform Service](/self-hosting/platform-service) for full configuration.
