The platform's backend logic lives in workspaces that need to be pushed to your Prisme.ai environment. The workspaces are stored as YAML files in the `workspaces/` directory.

## Deployment Order

Due to inter-workspace dependencies, deploy in this order:

<Steps>
  <Step title="Standalone workspaces">
    Deploy workspaces with no dependencies first:
    - `ai-governance-v2`
    - `vector-opensearch` (or your chosen vector provider)
    - `vector-mock` (optional, for testing)
    - `capabilities`
    - `prompt-library`
    - `parsers`
  </Step>
  <Step title="Service layer">
    Deploy services that depend on governance and vector providers:
    - `llm-gateway` (depends on governance)
    - `storage` (depends on llm-gateway, vector providers)
  </Step>
  <Step title="Tool workspaces">
    Deploy tools that depend on llm-gateway:
    - `tools-guardrails`
    - `tools-memories`
    - `tools-search-bing`
  </Step>
  <Step title="Product workspaces">
    Deploy products that depend on the full stack:
    - `agent-factory` (depends on governance, llm-gateway, storage, memories, guardrails)
    - `ai-collection-v3`
    - `ai-insights-v2` (depends on agent-factory)
    - `agent-evaluations` (depends on agent-factory)
  </Step>
</Steps>

## Push Workspaces

Using the Prisme.ai Builder MCP or CLI:

```bash
# Push a workspace to an environment
# (exact command depends on your tooling setup)
```

Or via the Builder UI at `/builder`:
1. Create a new workspace
2. Import the YAML files from `workspaces/<slug>/`

## Secrets Configuration

Each workspace needs environment-specific secrets configured:

### AI Governance
| Secret | Description |
|--------|-------------|
| `adminAccessToken` | Super admin token |

### LLM Gateway
| Secret | Description |
|--------|-------------|
| `OPENAI_API_KEY` | OpenAI API key |
| `AZURE_OPENAI_KEY` | Azure OpenAI API key |
| `ANTHROPIC_API_KEY` | Anthropic API key |
| (others per provider) | Provider-specific credentials |

### Agent Factory
| Secret | Description |
|--------|-------------|
| (references governance by slug) | Auto-configured via workspace config |

### AI Insights
| Secret | Description |
|--------|-------------|
| `AGENT_FACTORY_API_KEY` | API key for calling Agent Factory |

### Agent Evaluations
| Secret | Description |
|--------|-------------|
| `AGENT_FACTORY_API_KEY` | API key for calling Agent Factory |

### Bing Search
| Secret | Description |
|--------|-------------|
| Azure AD credentials | tenant_id, client_id, client_secret |

### Storage
| Secret | Description |
|--------|-------------|
| Vector provider auth | Auth header for OpenSearch/Elasticsearch |

## Environment-Specific Config

Workspace `index.yml` files contain environment-specific values that need updating:

| Setting | Location | Update For |
|---------|----------|-----------|
| `ai_governance_slug` | Agent Factory, LLM Gateway | Governance workspace slug |
| `ai_governance_id` | Agent Factory | Governance workspace ID |
| `default_vector_provider` | Storage | Your vector backend |
| `tools_memories_slug` | Agent Factory | Memories workspace slug |
| Provider endpoints | LLM Gateway | Your LLM provider endpoints |

## Verification

After deployment, run the workspace integration tests:

```bash
# Test each workspace
curl -X POST https://api.your-domain.com/v2/workspaces/slug:llm-gateway/webhooks/tests/run-all \
  -H "Authorization: Bearer $TOKEN"

curl -X POST https://api.your-domain.com/v2/workspaces/slug:storage/webhooks/tests/run-all \
  -H "Authorization: Bearer $TOKEN"
```

Check the [Testing](/development/testing) page for the full list of test suites.
