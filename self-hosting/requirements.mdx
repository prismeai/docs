To deploy Prisme.ai, your environment must meet these prerequisites.

## Infrastructure Recommendations

- **Nodes**: At least 3 Kubernetes nodes with 4 vCPU and 16GB RAM each
- **Disk Space**: Minimum 50GB persistent storage
- **High Availability**: Managed services or clusters with redundancy
- **Load Balancer**: Ingress controller or cloud load balancer for service routing

## Database Requirements

- **MongoDB or PostgreSQL**:
  - MongoDB 6.x+ or PostgreSQL 10+
  - 3-node replica set, 2GB RAM / 2 vCPU / 1,000 IOPS per node

- **Redis**:
  - Version 5.x+
  - Separate instances recommended for cache and broker
  - 3GB RAM and 2 vCPU per node

- **Elasticsearch/OpenSearch**:
  - Version 8.x+
  - 3-node cluster, 16GB RAM / 4 vCPU per node

### Vector Database

Required for AI Knowledge and Storage workspace features:

- **OpenSearch/Elasticsearch** — Can reuse the same cluster or a separate one
- **Redis Stack** — Redis 5.x+ with JSON and SEARCH modules, 3 nodes with 5GB RAM / 2 vCPU each

## Storage

Prisme.ai requires file or object storage for models and file uploads:

- **RWX Filesystem** — At least 50GB, attached via Kubernetes PVC
- **S3-Compatible Storage**
- **Azure Blob Storage**
- **Google Cloud Storage**

## Networking and DNS

- **Wildcard DNS** — Required for workspace routing (e.g., `*.pages.yourdomain.com`)
- **Dedicated DNS entries**:
  - `api.yourdomain.com` — API Gateway
  - `platform.yourdomain.com` — Platform frontend (new)

If wildcard DNS cannot be configured, create explicit entries for each product page.

## Authentication (SSO)

Prisme.ai supports multiple SSO options:

- **OIDC** — Google, Azure, Okta, Auth0, Keycloak, and generic providers
- **SAML** — HTTP-POST binding (Okta, ADFS, OneLogin, PingIdentity)
- **Microsoft Entra ID** — Specialized integration

See [Enterprise Authentication](/self-hosting/enterprise/authentication) for configuration details.

## Offline Environments

For air-gapped deployments:

- A npm registry reachable from the cluster
- Internal/self-signed TLS certificates must be configured in microservices
